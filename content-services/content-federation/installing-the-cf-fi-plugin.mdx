The Content Federation Services (CFS) application uses a CF Plugin to enable communication with the content repository. An OnBase content repository requires the CF Federated Identity (FI) Plugin to map users and their corresponding group associations to users in the Content Innovation Cloud (CIC).

CAUTION: Installing the CF Plugin is only required for OnBase repositories. The CF FI Plugin installation files are available to download from Hyland Community.
To install the CF FI Plugin:

1.  Navigate to the folder where the CF FI Plugin installation files were downloaded and open Hyland.ContentFederation.FederatedIdentity.Plugin.Installer.msi. The first page of the Content Federation FI Plugin installation wizard is displayed.
2.  Click Next. The End-User License Agreement page is displayed.
3.  After reading and accepting the terms in the license agreement, click Next. The Destination Folder page is displayed.
4.  To set the location for where the CF FI Plugin is installed, do one of the following:
- Enter the file location in the text field.
- Browse to the location by clicking Change.
By default, the CF FI Plugin is installed to the following location: C:\Program Files\Hyland\Hyland Content Federation Federated Identity Plugin.
5.  Click Next. The Ready to Install page is displayed.
6.  Click Install.
7.  After the installation is complete, click Finish.
8.  Configure the CF FI Plugin settings to establish communication with the content repository. See Configuring the CF FI Plugin.
9.  Create a new client in Hyland IdS to facilitate communication for user mapping. See Configuring Hyland IdS to Connect with the CF FI Plugin.
10.  Configure the CF Broker to coordinate communication with the CF FI Plugin and the content repository. See Coordinating the CF Broker and CF FI Plugin.
11.  Repeat the installation steps for each data source in your content repository.
