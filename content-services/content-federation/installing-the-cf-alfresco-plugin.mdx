---
title: "Installing The Cf Alfresco Plugin"
---

The Content Federation Services (CFS) application uses a CF Plugin to enable communication with the content repository. An Alfresco content repository requires the CF Alfresco Plugin.

CAUTION: Installing the CF Alfresco Plugin is only required for Alfresco repositories.
The CF Alfresco Plugin installation files are available to download from Hyland Community.

To install the CF Alfresco Plugin:

1.  From Hyland Community, download the following files:
| Operating system | File Name |
| --- | --- |
| Linux | alfresco-plugin-linux-x64-x.x.x.zip |
| Windows | alfresco-plugin-win-x64-x.x.x.zip |

2.  Extract from the .zip file and save the extracted files to a suitable location.
Linux example: /home/dev-admin/Documents/docker-container-test/Hyland Content Federation Alfresco Plugin/

Windows example: C:/Program Files/Hyland/Hyland Content Federation Alfresco Plugin

The Alfresco Plugin is installed.
3.  Create the following files and make sure they are saved to the same directory as the CF Alfresco Plugin installation files in step 2:

| File | Description |
| --- | --- |
| docker-compose.yml | Specifies the image details |
| docker-compose.override.yml | Overrides the main configuration with user-defined values |
| .env | Specifies environment variables and their corresponding values |

4.  Configure the docker-compose.yml, docker-compose.override.yml, and .env files with information based on your installation configuration. See Deploying CF Broker in a Container.
5.  Configure the CF Alfresco Plugin settings to establish communication with the content repository. See Configuring the CF Alfresco Plugin.
6.  Configure the CF Broker to coordinate communication with the CF Alfresco Plugin and the content repository. See Coordinating the CF Broker and CF Alfresco Plugin.
7.  Configure KeyCloak for token authentication. See Configure KeyCloak for token exchange with Alfresco.
8.  Repeat the installation steps for each Alfresco data source in your content repository.
