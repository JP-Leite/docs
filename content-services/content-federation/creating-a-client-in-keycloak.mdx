---
title: "Creating A Client In Keycloak"
---

Clients are entities that can request Keycloak to authenticate a user through an access token. For KeyCLoak to perform token exchange, a client must be created in the KeyCloak Administration Console.

To create a client in KeyCloak:

1.  Log in to the KeyCloak Administration Console.
2.  From the left pane, under Manage, click Clients. The Clients page is displayed.
3.  Click the Create client button. The Create client page is displayed.
4.  Under General Settings, from the Client Type list, select OpenID Connect.
5.  In the Client ID field enter the client ID.
6.  In the Name field enter the client name.
7.  In the Description field enter a brief decription of the client.
8.  In the Always display in UI field, click the switch to set the client visibility in the user interface from Off to On.
9.  Click Next. The Capability config screen appears.
10.  In the Client authentication field, click the switch to set the client authentication capability from Off to On.
11.  In the Authorization field, click the switch to set the client authorization capability from Off to On.
12.  In the Authentication flow field, select Standard Token Exchange and Service Account Roles.
13.   Click Next. The Login settings screen appears.
14.  Click Save. The client is created successfully and the Client details page is displayed.
15.  Click the Permissions tab. The Permissions screen is displayed.
16.  In the Permissions enabled field, click the switch to set the permission for the new client interface Off to On. The Permission list is displayed.
17.  From the Permisson list, select token-exchange.  The permission details are displayed.
18.  From the Policies list, select Create client policy.  The Choose a policy type wizard appears.
19.  Click Client. The Create client policy page appears.
20.  In the Name field, enter the name for the policy.
21.  In the Description field, enter a brief description of the policy.
22.  From the Clients list, select the client that was created.
23.  In the Logic field, select Positive.
24.  Click Save. The client policy is saved.Note:
Navigate to the Credentials tab of the newly-created client to find the Client Secret value which is needed for configuring the Alfresco Plugin.
