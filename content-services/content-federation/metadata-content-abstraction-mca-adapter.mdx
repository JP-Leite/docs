title: "Metadata Content Abstraction Mca Adapter"
---

For OnBase repositories, the Content Federation Metadata-Content Abstraction (CF MCA) Adapter component is required for CFS to interact with OnBase.

The CF MCA Adapter exposes a repository-agnostic content and metadata model. This allows client applications to interact with content and metadata in a repository and translate repository-specific concepts, such as Document Types in OnBase, into more generic concepts that can be applied across multiple platforms.

Note: Any changes to OnBase Configuration, such as associating a Document Type to a Document Type Group, needs the MCA Application Pool to be recycled before the changes are reflected in CFS. Recycling the Application Pool may be done manually through IIS Manager or by running the command iisreset.
For specific steps on installing and using the CF MCA Adapter, see the Content Federation Metadata and Content Abstraction documentation.

## Cryptography Compatibility

When using the CF MCA Adapter with OnBase, in OnBase Configuration, the Cryptography Settings must be configured so that Version Compatibility is set to Version 24 or lower for Hyland.FCC.Mca.Onbase.WebApi v24.1.0 or Version 25 or lower for Hyland.FCC.Mca.Onbase.WebApi v25.1.0. The Cryptography Settings are found under Utils > Cryptography. For more information on cryptography settings in OnBase, see Changing the Encryption Suite and Version Compatibility in the System Administration documentation.

CAUTION: If Version Compatibility is already set to a version greater than what is recommended, do not set Version Compatibility to a version lower than that set version. Contact Hyland Support for assistance.
