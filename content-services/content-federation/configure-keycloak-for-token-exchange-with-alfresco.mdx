KeyCloak integrates with the Content Federation Services (CFS) application as an external application. For Alfresco, KeyCloak is required to establish connection between Content Innovation Cloud (CIC) and Alfresco systems through token exchange.

Note:
Before configuring KeyCloak, ensure that the Legacy Token Exchange (version 1) is enabled for the keycloak server: kc.features = preview,token-exchange,admin-fine-grained-authz:v1.

Configuring KeyCloak for token exchange with Alfresco requires the following actions to be taken:

1.  Creating a Client in KeyCloak
2.  Creating an Identity Provider in KeyCloak
3.  Adding a User in KeyCloak
