## Accessing Cf Broker Error Logs

CF Broker error logs are written to the Hyland.ContentFederation.Broker.json file.

To access CF Broker error logs, navigate to and open the Hyland.ContentFederation.Broker.json file. By default, the file path for the error log is %PROGRAMDATA%/Hyland/logs/Hyland Content Federation Broker/Hyland.ContentFederation.Broker.json. The error log is displayed in JSON.  To change where the error log is stored and the minimum level of message recorded to the error log, edit the CF Broker's appsettings.json file. For more information, see Configuring the CF Broker.

## Accessing Cf Gateway Error Logs

The CF Gateway connects to your content repository through the CF Broker. The CF Gateway is part of the Content Innovation Cloud (CIC), so its error logs cannot be accessed without assistance from Hyland Support.

If there is an error with the CF Gateway, you may receive an error message from the CFS Consumer API. This error message contains the attribute ActivityId.

To access error logs for the CF Gateway:

1.  Record the ActivityId from the CFS Consumer API error message.
2.  Contact Hyland Support. The ActivityId is used to find the error log associated with your CF Gateway error.

## Accessing Cfs Content Connector Error Logs

Error logs for processes that involve CFS Content Connectors may be accessed in the Automate Administration application. Information within the error log may then be used to address the error.

For more information on the Automate Administration application, see the Hyland Automate Administration documentation.

To access error logs for CFS Content Connectors:

1.  Log in to the Automate Administration App.
2.  From the left pane, select Process Admin > Process Instances. A list of process instances is displayed.
3.  Locate the process associated with the CFS Content Connector that is experiencing errors. You may narrow the results in the list of process instances by using the filters above the list.
4.  Click the Options button and select Audit. A list of events related to the selected process instance is displayed.

The Event Type column displays the kind of event that occurred, including errors. You may use the filters above the event list to narrow the events to only errors. For more information on an individual error, click the link in the Event Action column to receive a full breakdown of the event in JSON.

## Checking Cfs Connections

The Content Federation Connection Manager allows you to check the status of CFS connections. Connections are tested automatically, and the status is updated each time the Content Federation Connection Manager page is loaded.

To check the status of CFS connections:
1.  Log in to the Content Innovation Cloud User Portal. For more information on how to log in to the Content Innovation Cloud User Portal, see Sign-in Options in the Content Innovation Cloud Administration Portal documentation.
2.  Ensure the appropriate environment is selected.
3.  Under Applications, click Content Federation Admin Portal. The Content Federation Connection Manager page is displayed. The Content Federation Connection Manager page shows a list of configured CFS connections with the most recently added connections at the top of the list. The connections are displayed with related information, including the Connection Name, Connection ID, Integration Name, Integration ID, and Status.
4.  For more information on a particular connection, such as the Plugin ID or (for SharePoint) the names of objects in the Query Object List, click the Actions button (![Actions button](data:image/gif;base64,R0lGODlhDQAZAHcAACH5BAAAAAAALAAAAAANABkAh////0dFUUdLbUdRmWRLUZFRUZHJ8cv5/O3Jmfn50Pn5/AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAhHABUIHEiwoMGDCBMqXEgQAQEBBhAmKBAgwIADBydWvIjQIUSGIBV6jJiRokWMBjWe7PiQZMiXDVtKNMkxJU2UBkfC3MmzYEAAOw==)) and select Connection Info. ## Checking Registered Brokers

The Content Federation Admin Portal allows you to see a list of installed CF Brokers in your environment.

To check registered brokers:

1.  Log in to the Content Innovation Cloud User Portal. For more information on how to log in to the Content Innovation Cloud User Portal, see Sign-in Options in the Content Innovation Cloud Administration Portal documentation.

3.  Under Applications, click Content Federation Admin Portal. The Content Federation Connection Manager page is displayed.

## Troubleshooting

The following topics describe how to find and resolve errors that you may see when using the Content Federation Services (CFS) application.

## V1X Setup Is No Longer Functioning

**Issue:**
Connections set up after installing the initial release version (v1.x) of Content Federation Services (CFS) no longer function, and logging in to the Content Federation Admin Portal shows a message at the top of the page that an update is required.

**Solution:**
Significant updates to the infrastructure of CFS have been made in the update to version 2. This introduces breaking changes that require users of CFS with setups from version 1 to update components in their existing solutions to be compatible with version 2.

For up-to-date information on how to upgrade an existing setup from version 1 to be compatible with version 2, see the [Upgrading to CFS 2.0](https://hyland.atlassian.net/wiki/spaces/HFCH/pages/3456925714/INTERNAL+-+Upgrading+to+CFS+2.0) page available on Confluence.
