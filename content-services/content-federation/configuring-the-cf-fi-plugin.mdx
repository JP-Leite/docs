After the CF FI Plugin is installed, the CF FI Plugin application settings need to be configured to communicate with the content repository.

CAUTION: When upgrading the CF FI Plugin to a new version, values in the PluginConfig.json file are not retained, so you need to reconfigure parameters in the file. For this reason, save a backup of the values in the PluginConfig.json file before upgrading the CF FI Plugin.
To configure the CF FI Plugin:

1.  Navigate to the folder where the CF FI Plugin is installed. By default, the folder is located in C:\Program Files\Hyland\Hyland Content Federation Federated Identity Plugin.
2.  Open the PluginConfig folder.
3.  Open the PluginConfig.json file in a plain-text editor, such as Notepad.
4.  Set values for the following parameters:

| PluginConfig.json Parameter | Value |
| --- | --- |
| IntegrationId | The value obtained from the Integration ID field during the registration of CFS in the Content Innovation Cloud Administration Portal For more information, see Adding a System Integration. |
| ScimEndpoint | The URL of the OnBase API Server SCIM service. For example, https://hostname.example.com/<ApiServerApplicationName>/onbase/SCIM/v2 |

5.  Under IdentityProviderOptions, enter information that corresponds to the Hyland Identity Service (IdS) identity provider for the OnBase data source. The following table describes the information needed for each parameter.  Note: Some information in the IdentityProviderOptions is related to Hyland IdS. For more information on Hyland IdS, see Hyland Identity Service (IdS) in the Identity and Access Management Services documentation.

| IdentityProviderOptions Parameter | Value to Enter |
| --- | --- |
| Address | The Hyland IdS URLFor example, https://hostname.example.com/<IdSApplicationName> |
| ClientId | The client identifier created in Hyland IdS for authenticating with the OnBase SCIM serviceNote: The client configured in the ClientId parameter must whitelist all of the scopes listed in the Scopes parameter to successfully perform the token exchange. |
| ClientSecret | The secret of the client identifier created in Hyland IdS |
| Scope | A space-separated list of the scopes required to authenticate with the OnBase SCIM service.For example, iam.user-catalog iam.user-catalog.read |

6.  Save and close the PluginConfig.json file.
