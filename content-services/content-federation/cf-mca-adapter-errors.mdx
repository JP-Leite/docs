The following table lists known errors related to the CF Broker interacting with the Content Federation Metadata-Content Abstraction (CF MCA) Adapter and how to resolve the errors.

Tip: It is recommended that you keep the Diagnostics Console open on the computer where the CF MCA Adapter Application Pool is running. This allows you to see the root cause of the error in the error logs under the McaOnBaseWebApi tab.
-
-

| Error Message | Cause | Resolution |
| --- | --- | --- |
| System.Threading.Tasks.TaskCanceledException: The request was canceled due to the configured HttpClient.Timeout of 100 seconds elapsing. | An HTTP exception when the user tries to retrieve data after any application pool is recycled once or multiple times. | Trigger the specific Consumer API again and wait for the response. |
| "Message":"[FileService.cs : DownloadFileFromPresignedUrlAsync] File download failed","Details":"StatusCode: NotFound, Reason: Not Found" | When using the CREATE_CONTENT action, the presigned URL used in the request payload is invalid. | Ensure the presigned URL is correct. You may output a presigned URL using the GET_CONTENT_PARTS action of the Content Federated Connector. |
| "Message":"[FileService.cs : DownloadFileFromPresignedUrlAsync] File download failed","Details":"StatusCode: Forbidden, Reason: Forbidden" | When using the CREATE_CONTENT action, the presigned URL used in the request payload has expired. | Ensure the presigned URL is up to date. You may output a presigned URL using the GET_CONTENT_PARTS action of the Content Federated Connector. |
| "Exception":"System.Net.Http.HttpRequestException: The SSL connection could not be established, see inner exception.\r\n ---> System.Security.Authentication.AuthenticationException: The remote certificate is invalid according to the validation procedure: RemoteCertificateNameMismatch, RemoteCertificateChainErrors | The CF Broker cannot validate the MCA Server’s certificate while calling any MCA Web API. | Try each of the following: On the computer where the CF Broker is running, export the self-signed certificate into the instance where the CF Broker is experiencing the error. Use a Root Certificate Authority that is trusted across your entire domain. |
