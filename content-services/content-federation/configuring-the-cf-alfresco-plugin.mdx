After the CF Alfresco Plugin is installed, the CF Alfresco Plugin application settings need to be configured to communicate with the content repository.

CAUTION: When upgrading the CF Alfresco Plugin to a new version, values in the PluginConfig.json file are not retained, so you need to reconfigure parameters in the file. For this reason, save a backup of the values in the PluginConfig.json file before upgrading the CF Alfresco Plugin.
To configure the CF Alfresco Plugin:

1.  Navigate to the folder where the CF Alfresco Plugin is installed.
2.  Open the PluginConfig folder.
3.  Open the PluginConfig.json file in a plain-text editor, such as Notepad.
4.  Set values for the following parameters:

| PluginConfig.json Parameter | Value |
| --- | --- |
| IntegrationId | The value obtained from the Integration ID field during the registration of CFS in the Content Innovation Cloud Administration Portal For more information, see Adding a System Integration. |
| AlfrescoApiUrl | The base URL of Alfresco Content Services instance being connected. |
| RepositoryId | Unique identifier of the Alfresco Content Repository. Enter value: -default- |
| RepositoryName | Display name for the repository. Enter value: alfresco |
| FolderId | The target folder node ID in Alfresco where you want to create content.  Note: If this field is empty, the contents are created in the root folder. |
| SearchMaxPageSize | The Upper limit that the user wants to set while searching for contents in Alfresco. By default the value is 2000 in the configuration file. |
| ExcludedAspects | A List of aspects in Alfresco that user wants to exclude from GetContentTypeById or GetContentPropertiesById actions.  By default the cm:auditable aspect is mentioned in the configuration file. |

5.  Under IdentityProviderOptions, enter information that corresponds to the KeyCloak identity provider for the Alfresco data source. The following table describes the information needed for each parameter. For more information on KeyCloak configuration, see Configure KeyCloak for token exchange with Alfresco.

| IdentityProviderOptions Parameter | Value to Enter |
| --- | --- |
| IdentityServerUrl | The Alfresco KeyCloak Provider URL.  For example https://hostname.example.com/auth/realms/<realmName>/protocol/ |
| ClientId | The client identifier created in KeyCloak for authenticating the Alfresco APIs. |
| ClientSecret | The secret of the client identifier created in KeyCloak. |
| GrantType | The value urn:ietf:params:oauth:grant-type:token-exchange, which allows for exchanging an Content Innovation Cloud token with an Alfresco access token. |
| SubjectTokenType | The value urn:ietf:params:oauth:token-type:jwt, which represents the type of token that is being exchanged. |
| RequestedTokenType | The value urn:ietf:params:oauth:token-type:access_token, which represents the type of token being requested from Alfresco. |

6.  Save and close the PluginConfig.json file.
