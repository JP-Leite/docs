---
title: "Hyland Ids Errors"
---

The following table lists known errors related to the CF Broker interacting with the Hyland Identity Service (IdS) and how to resolve the errors.

Tip: It is recommended that you keep the Diagnostics Console open on the computer where the Hyland IdS Application Pool is running. This allows you to see the root cause of the error in the error logs under the Hyland.Identity.Service tab.

| Error Message | Cause | Resolution |
| --- | --- | --- |
| System.Exception: Error while fetching token: StatusCode: Bad Request, Message: {\"error\":\"invalid_grant\”} | An error exchanging the HxIAM token for the Hyland IdS token. | Try each of the following: Check the following values in the Hyland IdS token exchange provider: Issuer Username claim JWKS URI   In the HxIAM token, check the preferred_username value. Ensure that the user listed in the value is present and has the correct permissions.  If you made any changes based on the above recommendations, perform an IIS Reset after making the changes. |
| System.Threading.Tasks.TaskCanceledException: The request was canceled due to the configured HttpClient.Timeout of 100 seconds elapsing. | An HTTP exception when the user tries to retrieve data after any application pool is recycled once or multiple times. | Trigger the specific Consumer API again and wait for the response. |
| Error while fetching token: StatusCode: Bad Request, Message: {\"error\":\"invalid_client\"} | An error exchanging the HxIAM token for the Hyland IdS token. | Check if the following values are configured correctly in the PluginConfig.json file:  ClientSecret: Ensure the client secret is not expired.  GrantType: Ensure that "Token Exchange" is an allowed grant type by entering urn:ietf:params:oauth:grant-type:token-exchange.  Scope: Ensure mca evolution is entered. |
| "Exception":"System.Net.Http.HttpRequestException: The SSL connection could not be established, see inner exception.\r\n ---> System.Security.Authentication.AuthenticationException: The remote certificate is invalid according to the validation procedure: RemoteCertificateNameMismatch, RemoteCertificateChainErrors | The CF Broker cannot validate the certificate of the Hyland IdS while performing a token exchange. This is due to the connection being marked as not secure. | Try each of the following: On the computer where the CF Broker is running, export the self-signed certificate into the instance where the CF Broker is experiencing the error. Use a Root Certificate Authority that is trusted across your entire domain. |
