---
title: "Configuring The Cf Onbase Plugin"
---

After the CF OnBase Plugin is installed, the CF OnBase Plugin application settings need to be configured to communicate with the content repository.

CAUTION: When upgrading the CF OnBase Plugin to a new version, values in the PluginConfig.json file are not retained, so you need to reconfigure parameters in the file. For this reason, save a backup of the values in the PluginConfig.json file before upgrading the CF OnBase Plugin.
To configure the CF OnBase Plugin:

1.  Navigate to the folder where the CF OnBase Plugin is installed. By default, the folder is located in C:\Program Files\Hyland.
2.  Open the PluginConfig folder.
3.  Open the PluginConfig.json file in a plain-text editor, such as Notepad.
4.  Set values for the following parameters:

| PluginConfig.json Parameter | Value |
| --- | --- |
| McaServerUrl | The Server URL of the MCA Adapter For example, https://hostname.example.com/``<McaApplicationName>``/ |
| IntegrationId | The value obtained from the Integration ID field during the registration of CFS in the Content Innovation Cloud Administration Portal For more information, see Adding a System Integration. |

5.  Optional: Configure MCA session cookie cleanup by editing the values of the EnableOnBaseSessionCookieCleanUp and OnBaseSessionCookieHeader parameters.
Each time a user makes a request to OnBase, MCA stores a cookie in the OnBase database. If frequent requests are made, the resulting large number of cookies could affect the performance of CFS. Enabling MCA session cookie cleanup prevents the buildup of cookies that result from MCA requests.

The following table outlines the settings related to MCA session cookie cleanup:

| PluginConfig.json Parameter | Description | Default Value |
| --- | --- | --- |
| EnableOnBaseSessionCookieCleanUp | Set the value to true to automatically remove the cookies that result from a request to OnBase.Setting the value to false or leaving the value empty leaves the cookies that are generated by requests to OnBase. | true |
| OnBaseSessionCookieHeader | Set the value to what you want to appear in the Set-Cookie header for any MCA HTTP operation. | Cookie.Session.OnBase.Hyland |

6.  Under IdentityProviderOptions, enter information that corresponds to the Hyland Identity Service (IdS) identity provider for the OnBase data source. The following table describes the information needed for each parameter.  Note:
Some information in the IdentityProviderOptions is related to Hyland IdS and the CF MCA Adapter.

For more information on Hyland IdS, see Hyland Identity Service (IdS) in the Identity and Access Management Services documentation. For more information about the CF MCA Adapter, see the Content Federation Metadata and Content Abstraction documentation.

| IdentityProviderOptions Parameter | Value to Enter |
| --- | --- |
| Address | The Hyland IdS URLFor example, https://hostname.example.com/<IdSApplicationName> |
| ClientId | The client identifier created in Hyland IdS for authenticating with the CF MCA AdapterNote: The client configured in the ClientId parameter must whitelist all of the scopes listed in the Scopes parameter to successfully perform the token exchange. |
| ClientSecret | The secret of the client identifier created in Hyland IdS |
| GrantType | The value urn:ietf:params:oauth:grant-type:token-exchange, which allows for exchanging an Content Innovation Cloud token with an OnBase token |
| SubjectTokenType | The value urn:ietf:params:oauth:token-type:access_token, which represents the type of token that is being exchanged |
| Scope | The value mca evolution, which represents a space-separated list of the scopes required to connect to the CF MCA Adapter and the Temporary File Service (TempFS) |
| IdPTenant | The Id or the Tenant Name of the tenant used by Hyland IdS to connect to OnBaseThe Id value may be found in the tenant information in the idsconfig.json file for Hyland IdS. By default, this file is located in C:\Program Files\Hyland\identityservice\config. The Tenant Name value may be found in the Hyland IdS Administration client. |

7.  Save and close the PluginConfig.json file.
