To better understand the Content Federation Services (CFS) application, this topic goes over what a CFS Connection is, what a Content Federated connector is, the available Content Federated connector configurations, and how these features interact with the CFS Consumer API.

** CFS Connections**
The CFS uses connections (CFS Connections) that are configured in the Content Federation Admin Portal. CFS Connections link the Content Innovation Cloud (CIC) and a specific content repository instance deployed within the customer's data center.
**Content Federated Connector**
The Content Federated connector is a component within Automate that uses a CFS Connection to perform actions on content and data within the customer's content repository instance. A Content Federated connector is applied to and configured in Automate Modeler to work with automation solutions. For more information on Automate processes, modeling, and use of connectors, see the Hyland Automate Modeler documentation.
There are 2 ways of configuring the Content Federated connector: on behalf of (OBO) an end user and through a service user (SU).
-  On Behalf Of an end user (OBO): When connecting OBO, the Content Federated connector enables running connector actions on behalf of an end user. This means that that user’s authorization rules are applied within the customer’s content repository. The CF Broker creates an audit record indicating that the content repository is being accessed on behalf of a specific end user. An HxP End User is mapped to a corresponding content repository end user. The audit record will contain both user identifiers. The Content Federated connector can be configured to assume the end user identity from any Automate user task executed earlier in the Automate process flow.
-  Through a Service User (SU): When connecting SU, the Content Federated connector enables running connector actions under the identity of an HxP Service Account that is mapped to a corresponding Service User Account within the customer’s target repository. The authorization permissions are applied as per mapped Service User Account settings. The audit record will indicate content repository access under the service account identity. Both CIC and repository System User Accounts will be included in the audit record.

** CFS Consumer API**
The CFS Consumer API provides CIC solutions with a set of content operations that can be performed over a CFS Connection. A Content Connector provides a low-code way to use the CFS Consumer API to perform operations.
