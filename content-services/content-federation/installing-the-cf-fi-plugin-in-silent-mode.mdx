---
title: "Installing The Cf Fi Plugin In Silent Mode"
---

You can use the Hyland.ContentFederation.FederatedIdentity.Plugin.Installer.msi to perform a silent installation by using the command prompt. Silent installation means that the installer runs without the GUI and without the need for user interaction.

The CF Federated Identity (FI) Plugin installation files are available to download from Hyland Community.
To install the CF FI Plugin in silent mode:

1.  Open Command Prompt with the Run as administrator option. The Command Prompt window is displayed.
2.  Navigate to the folder where the CF FI Plugin installation files were downloaded.
3.  Execute the following command to install the CF FI Plugin in silent mode:
```
msiexec /i "Hyland.ContentFederation.FederatedIdentity.Plugin.Installer.msi" /q ACCEPT_EULA=1
```

The following table describes the switches within the command:

| Command Switch | Description |
| --- | --- |
| /i | Triggers the installation |
| /q   /silent   /quiet | Triggers the silent installation |
| ACCEPT_EULA | Confirms End User License Agreement (EULA) acceptance during installation. Setting this switch to 1, Y, y, YES, yes, TRUE, or true accepts the EULA. |

4.  After the installer has completed, perform the following configuration steps:
1.  Configure the CF FI Plugin settings to establish communication with the content repository. See Configuring the CF FI Plugin.
2.  Create a new client in Hyland IdS to facilitate communication for user mapping. See Configuring Hyland IdS to Connect with the CF FI Plugin.
3.  Configure the CF Broker to coordinate communication with the CF FI Plugin and the content repository. See Coordinating the CF Broker and CF FI Plugin.
5.  Repeat the installation steps for each data source in your content repository.
