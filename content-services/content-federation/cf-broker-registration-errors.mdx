| Error Message | Cause | Resolution |
| --- | --- | --- |
| System.Net.Http.HttpRequestException: No connection could be made because the target machine actively refused it. | The GatewayApiServerUrl value uses the "HTTP" protocol instead of "HTTPS". | From the CF Broker installation directory, open appsettings.json. Under GatewayConfiguration, locate the GatewayApiServerUrl parameter. Ensure that the URL in the GatewayApiServerUrl value begins with https. |
| Hyland.ContentFederation.Broker.KeyServices.Exceptions.RegistrationException: Failed to call registration endpoint. Status code: InternalServerError, response: {\"message\": \"Internal server error\"} | The gateway signing and registration keys used in the CF Broker configuration may be invalid. | From the CF Broker installation directory, open appsettings.json. Under GatewayConfiguration, locate the GatewaySigningKey and GatewayRegistrationKey parameters. Ensure that the values in GatewaySigningKey and GatewayRegistrationKey are correct. |
| System.Net.Http.HttpRequestException: No such host is known. | The GatewayApiServerUrl value contains an invalid hostname. | From the CF Broker installation directory, open appsettings.json. Under GatewayConfiguration, locate the GatewayApiServerUrl parameter. Ensure that the URL in the GatewayApiServerUrl value is correct and contains a valid hostname. |
| System.IO.DirectoryNotFoundException: Could not find a part of the path ‘C:\\protectedkey\\keys.json’ | The ProtectedKeyFilePath value is wrong or doesn't exist. | From the CF Broker installation directory, open appsettings.json. Under GatewayConfiguration, locate the ProtectedKeyFilePath parameter. Ensure that the ProtectedKeyFilePath value is present and correct. |
| Hyland.ContentFederation.Broker.KeyServices.Exceptions.ConfigurationException: Failed to decrypt BrokerPublicKey | The BrokerPublicKey value in the Protected Key File is invalid. | Stop the CF Broker service. Delete the Protected Key File. Run the CF Broker service. |
| Hyland.ContentFederation.Broker.KeyServices.Exceptions.ConfigurationException: Failed to load configuration from file.\r\n ---> System.Text.Json.JsonException: The JSON value could not be converted to Hyland.ContentFederation.SignAndVerify.Payload.Enums.SignatureTypesEnum. Path: $.gatewayKeyType | The gatewayKeyType value in the Protected Key File is invalid. | Stop the CF Broker service. Delete the Protected Key File. Run the CF Broker service. |
| System.UnauthorizedAccessException: Access to the path 'C:\Program Files (x86)\Hyland\keys.json' is denied. at Microsoft.Win32.SafeHandles.SafeFileHandle.CreateFile(String fullPath, FileMode mode, FileAccess access, FileShare share, FileOptions options) | The CF Broker service doesn’t have necessary access to the Protected Key File path. | Give Read/Write access for the Protected Key File path to the LOCAL SERVICE account. |
| System.ArgumentException: BrokerRegistrationToken cannot be null or empty. (Parameter 'token') | No token is provided in BrokerRegistrationToken parameter. | From the CF Broker installation directory, open appsettings.json. Under GatewayConfiguration, locate the BrokerRegistrationToken parameter. Ensure that the BrokerRegistrationToken value contains a valid token.  For more information on how to generate a token for the CF Broker, see Generating Broker Tokens. |
| Hyland.ContentFederation.Broker.KeyServices.Exceptions.RegistrationException: Failed to call registration endpoint. Status code: BadRequest, response: {"title":"Bootstrapping failed.","status":400,"detail":"The provided bootstrap JWT token is invalid.","eventId":8000,"activityId":"072e219a-08bc-4564-a391-364259d0fed1"} | The BrokerRegistrationToken parameter has an expired or invalid token value. | From the CF Broker installation directory, open appsettings.json. Under GatewayConfiguration, locate the BrokerRegistrationToken parameter. Ensure that the BrokerRegistrationToken value contains a valid token.  For more information on how to generate a token for the CF Broker, see Generating Broker Tokens. |
| Hyland.ContentFederation.Broker.KeyServices.Exceptions.RegistrationException: Failed to call registration endpoint. Status code: Forbidden | The HXIAM client credentials being used to call the CF Broker registration endpoint do not have sufficient permissions. | Try each of the following: From the CF Broker installation directory, open appsettings.json. Under Credentialsensure the ClientId and ClientSecret values are correct. Generate an access token with the ClientId and ClientSecret to check if the associated client has the cf-api-gateway.cf-broker-bootstrapper role assigned. Ensure that the CF Broker service users are assigned the correct user roles. For more information, see Assigning User Roles to CF Broker User Groups. |
