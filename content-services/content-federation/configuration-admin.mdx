## Adding A New Cfs Connection

The Content Federation Connection Manager may be used to add new connections after the initial installation of the Content Federation Services (CFS) application. New connections are added through the Content Federation Admin Portal.

To add a new CFS connection:

1.  Log in to the Content Innovation Cloud User Portal. For more information on how to log in to the Content Innovation Cloud User Portal, see Sign-in Options in the Content Innovation Cloud Administration Portal documentation.
2.  Ensure the appropriate environment is selected.
3.  Under Applications, click Content Federation Admin Portal. The Content Federation Connection Manager page is displayed.
4.  Click Add new connection. The New Connection wizard is displayed.
5.  From the Select Integration field, select the system integration that corresponds to the software being used by the data source, such as OnBase.  Note: The options in the Select Integration field are created in the Content Innovation Cloud Administration Portal. For information on adding system integrations, see Adding a System Integration.
6.  Click Next. The Configuration page is displayed.
7.  In the Connection Name field, enter a unique, meaningful name for the connection.
8.  In the Description field, enter a description of the connection.
9.  Enter information in the following fields depending on the system integration selected in step 5.
- If you selected an OnBase, Alfresco, or Nuxeo integration, skip to step 10.
- If you selected a SharePoint integration, enter information in the following fields:Note: The information required for the following fields includes information accessible through Microsoft Azure. For more information on where and how to obtain this information, see Acquiring Connection Information for SharePoint.

| Field | Value |
| --- | --- |
| Client ID | The Application (client) ID of the registered external application used by SharePoint The Application (client) ID value may be found in Microsoft Azure. |
| Client Secret | The Secret associated with the registered external application used by SharePoint The Client Secret is created for a given Client ID in Microsoft Azure. |
| Tenant | The Directory (tenant) ID of the tenant used by SharePoint The Directory (tenant) ID value may be found in Microsoft Azure. |
| User Group | The ID of the user group containing all CIC users that need access to the documents in SharePoint |
| Site ID | A unique identifier for the SharePoint site |
| Query Object List | A list of SharePoint List IDs that are able to be queried or configured by CFS |

10.  Click Create Connection. The Content Federation Connection Management page is displayed, and the new connection is available in the list of connections.
11.  Check the newly configured CFS Connection to make sure it is connecting successfully. For more information, see Checking CFS Connections.

## Adding A System Integration

The Content Federation Services (CFS) application integrates with the software being used by the data source, such as OnBase. To integrate with the data source's software, CFS needs to have a system integration added and configured in the Content Innovation Cloud Administration Portal.

To add a system integration:

1.  Log in to the Content Innovation Cloud Administration Portal. For more information on how to log in to the Content Innovation Cloud Administration Portal, see Sign-in Options in the Content Innovation Cloud Administration Portal documentation.
2.  Select the External Systems tab under the banner.
3.  In the sidebar, click System Integrations. The System Integrations page is displayed.
4.  Click Add System Integration. The Add System Integration dialog box is displayed.
5.  In the Name field, enter a name for the system integration.
6.  From the System drop-down menu, select the software being used by the data source. You can choose from OnBase, Alfresco, and Nuxeo.  Note: A new integration needs to be created for each data source.
7.  From the Environment drop-down menu, select the environment to which you would like to add the system integration.
8.  Click Add System Integration. The system integration is created and available for use with CFS connections.

## Adding A User In Keycloak

1.  Log in to the KeyCloak Administration Console.
2.  From the left pane, under Manage, click Users.
3.  Click Add user. The Create user page is displayed.
4.  Type the details, as required.
5.  Click Create. The User details page of the newly created user is displayed.
6.  To link the user with an identity provier, click the Identity provider links tab. A list of available identity providers is displayed.
7.  Click Link Account next to the identity provider which needs to be linked with the user. The Link account wizard is displayed.
8.  In the User Id field, type sub, which is the subject claim value of incoming HxIAM Token.
9.  In the Username field, type the preferred_username claim value of the incoming HxIAM Token.
10.  Click Link. The identity provider is linked with the user.Note: The user needs to have the appropriate permissions to access any folder in the Alfresco repository. For more information, see Accessing a Folder in an Alfresco Repository.

## Additional Secure Environment Recommendations

It is recommended that the Content Federation Services (CFS) components are installed using the following additional recommendations to help secure the installation environment. These recommendations should be followed as a minimum requirement for the Content Federation Services. The policies of your organization may have additional or more robust requirements that should also be followed.

The recommendations listed in this topic are in addition to the default recommendations that apply to all Hyland products. For details on the default recommendations for all Hyland products, see Secure Environment Recommendations.

- The CF Broker and CF Plugin should be deployed into a dedicated network segment. This should be separate from the segments where the OnBase system is deployed.
- The network segment where the CF Broker is deployed has to be configured with HTTPS access to the OnBase MCA Adapter and Hyland IdS. This traffic must be contained within the customer data center.
- The CF Broker should not be allowed to make direct calls on any other OnBase components.
- The CF Broker must be able to make HTTPS calls into the Content Innovation Cloud (CIC).
- Incoming HTTPS traffic to the CF Broker network segment is not needed. It is recommended that incoming HTTPS traffic to the CF Broker is disabled.
- For the CFS to work, OnBase does not need to have the ability to call into CIC. If OnBase is already deployed in a way that does not allow it to make calls outside of the customer data centers, this configuration does not need to change.
- Sensitive values in the CFS configuration files have to be encrypted using the Hyland Application Settings Utility. For more information, see Hyland Application Settings Utility in the API Server documentation.
- For CIC communications from the broker:
- Define a list of protocols and ports.
- Define File System permissions to restrict updates to the installation directory to customer system administrators.
- Define File System permissions to restrict access to generated log files. Restrict access to only system administrators that require access to perform their duties.
- Define the Windows service account that the broker is running as to restrict system access.
- When configuring CFS Connector (Service User) instances, configure one instance per Automate task with a dedicated CIC Service User and a corresponding dedicated OnBase Service User for each CFS Content Connector task within an Automate process. Set authorization permissions to the minimum necessary required to execute a given task.

## Configuration Values

Configuration values can be supplied in 3 different ways:

- Environment variables
- KeyPerFiles
- Configuration file

If the container is configured with multiple configuration values, the value that is used is chosen in this order of precedence: Environment Variables > KeyPerFiles > Configuration file.

Note: The options in the Select Integration field are created in the Content Innovation Cloud Administration Portal. For information on adding system integrations, see Adding a System Integration.

## Methods Of User Configuration

There are 2 ways of configuring the Content Federated connector: on behalf of (OBO) an end user and through a service user (SU).

**On Behalf Of an end user (OBO)**
When connecting OBO, the Content Federated connector enables running connector actions on behalf of an end user. This means that that user’s authorization rules are applied within the customer’s content repository. The CF Broker creates an audit record indicating that the content repository is being accessed on behalf of a specific end user. An HxP End User is mapped to a corresponding content repository end user. The audit record will contain both user identifiers. The Content Federated connector can be configured to assume the end user identity from any Automate user task executed earlier in the Automate process flow.
**Through a Service User (SU)**
When connecting SU, the Content Federated connector enables running connector actions under the identity of a CIC Service Account that is mapped to a corresponding Service User Account within the customer’s target repository. The authorization permissions are applied as per mapped Service User Account settings. The audit record will indicate content repository access under the service account identity. Both CIC and repository System User Accounts will be included in the audit record.
A Content Federated connector instance is configured as part of a project in Automate, and then it is deployed as part of a process to Hyland Workspace. It is possible to create multiple Content Federated connector instances within a single Automate process. It is recommended to create a dedicated Content Federated connector instance for every task within a given Automate process.

Note: For OnBase users, if configuring the Content Federated connector SU, it is important to ensure that a corresponding OnBase Service User account is created in OnBase for each Automate task that uses the Content Federated connector. Each corresponding OnBase Service User needs to have the minimum access necessary to perform the associated Automate task.

## Secure Environment Recommendations

The following recommendations are intended to help secure the installation environment and apply to all product installations. These recommendations should be followed as a minimum requirement for all Hyland products. The policies of your organization may have additional or more robust requirements that should also be followed.

Hyland products may have additional recommendations described in the specific documentation for that product. In some cases, the recommendations may change or may only apply when using certain Hyland products together in a solution.

- Use TLS for all HTTP traffic, including private network segments. TLS ciphers have to be maintained to stay current over time.
- Use Secure FTP instead of standard FTP for all FTP traffic.
- End-to-end encryption is recommended for all data in transport, independent of a network segment. Note that some regulatory compliance requirements may require end-to-end encryption.
- Change all default passwords before activation of the production system. This applies to Hyland products as well as third-party products used by Hyland products (such as a database server).
- Authorization rules should be configured and tested before activation of the production system. This applies to Hyland products as well as to file system folders and database user accounts.
- Use database encryption for all sensitive data persisted in the database.
- Use file system encryption for all sensitive data and content persisted on the file system.
- Enable encryption when available as part of a subsystem configuration. For example, since ODBC provides the capability to use strong encryption for data, it is recommended to have that option enabled.

## Side-Channel Risks

Consider the following to mitigate side-channel risks:

- Ensure the latest application and operating system patches are applied.
- Ensure the latest firmware patches are applied for any hardware on-premises.

## User Mapping

The Content Federation Services (CFS) application allows users to map user accounts from OnBase to matching user accounts in the Content Innovation Cloud (CIC). This allows agents in Knowledge Discovery to access accurate OnBase user information, such as User Groups, and it ensures that CIC users are only able to read or view OnBase content to which they have access. Mapping OnBase users to CIC users is available on the User Mapping tab, which is part of the Content Federation Admin Portal.

The following topics describe how to perform actions on the User Mapping tab:

-  Enabling User Mapping
-  Disabling User Mapping
