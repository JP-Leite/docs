A mixin is a data structure that allows a document to contain multiple schemas and their associated fields without having to inherit another document with these schemas. By allowing schemas and fields to be injected into, rather than inherited by, a content type, mixins promote the reuse of these features. Additionally, a mixin can be added dynamically to a document instance, even if the content type does not reference the mixin.
