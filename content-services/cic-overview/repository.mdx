The **repository** is the central content store within the Content Innovation Cloud. It stores, organises, and serves all documents and their associated metadata.

## How It Works

The repository functions similarly to a database — it embeds security logic and availability constraints directly into the document storage layer, rather than relying purely on application-level controls. This ensures that content is protected and accessible regardless of which CIC application or API is used to access it.

Key characteristics:

- **Hierarchical organisation** — Documents are arranged in a folder tree, and permissions are inherited down the hierarchy.
- **Embedded security** — ACL and permission checks are enforced at the repository layer, not just the application layer.
- **Metadata-rich storage** — Every document carries typed, searchable metadata defined by its content type.
- **Version awareness** — The repository natively tracks document versions and working copies.

<Info>
  The repository is the foundation on which all CIC applications — Workspace, Studio Modeler, Knowledge Discovery, and the REST API — are built. Understanding the repository model is essential for both developers and administrators.
</Info>
