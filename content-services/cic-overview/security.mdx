## Security Model Overview

Content Innovation Cloud employs a layered security model to protect content at every level of the repository hierarchy.

<CardGroup cols={2}>
  <Card title="Global Permissions" icon="globe" href="/content-services/cic-overview/global-permissions">
    Role-based permissions applied globally across the repository. Determined by the user's assigned role in their CIC account.
  </Card>
  <Card title="Repository Permissions" icon="database" href="/content-services/cic-overview/repository-permissions">
    Permissions that control access to specific areas of the repository — complementing and sometimes restricting global permissions.
  </Card>
  <Card title="ACL Permissions" icon="lock" href="/content-services/cic-overview/acl-permissions">
    Fine-grained, document-level access control lists (ACLs) that define exactly which users or groups can perform which actions on a specific document.
  </Card>
  <Card title="Secure Environment" icon="shield-halved" href="/content-services/cic-overview/secure-environment-recommendations">
    Best-practice recommendations for securing your CIC installation environment, including TLS, encryption, and password policies.
  </Card>
</CardGroup>

## How Permissions Interact

Permissions in CIC are evaluated in layers:

<Steps>
  <Step title="Global Role Check">
    The user's **global role** (e.g., System Administrator, Business User) determines the broadest set of actions they can perform across the repository.
  </Step>
  <Step title="Repository Permission Check">
    **Repository-level permissions** refine access, scoping the user's rights to specific content areas.
  </Step>
  <Step title="ACL Permission Check">
    **Document-level ACLs** provide the most granular control. ACLs can grant or restrict access on individual documents, overriding higher-level permissions where configured.
  </Step>
</Steps>

<Info>
  ACL permissions are **inherited through the folder hierarchy**. A permission set on a parent folder is automatically inherited by all child documents and folders unless explicitly overridden.
</Info>

## Secure Environment Recommendations

Key security practices for all CIC installations:

| Practice | Recommendation |
| --- | --- |
| **Transport encryption** | Use TLS for all HTTP traffic, including private network segments |
| **File transfer** | Use Secure FTP (SFTP) instead of standard FTP |
| **Default passwords** | Change all default passwords before going live |
| **Database encryption** | Encrypt all sensitive data at rest in the database |
| **File system encryption** | Encrypt sensitive content on the file system |
| **Patch management** | Apply the latest OS, application, and firmware patches promptly |
