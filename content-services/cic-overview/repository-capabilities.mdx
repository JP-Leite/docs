## Overview

The Hyland Content repository provides a rich set of capabilities that go beyond simple file storage. These capabilities are the foundation for everything you can build on top of CIC.

---

## Core Capabilities

<CardGroup cols={2}>
  <Card title="Upload & Download" icon="arrow-up-from-bracket" href="/content-services/cic-overview/upload-and-download">
    Ingest content into the repository via REST API or the Workspace UI. Download single documents or batches with full binary fidelity.
  </Card>
  <Card title="Search (HXQL)" icon="magnifying-glass" href="/content-services/cic-overview/search">
    Query content using **HXQL** — Hyland's SQL-like query language — against document metadata and full-text content extracted by the Transformation Service.
  </Card>
  <Card title="Versioning" icon="code-branch" href="/content-services/cic-overview/version">
    Maintain a complete version history for every document. Restore previous versions or compare changes over time.
  </Card>
  <Card title="Working Copy" icon="copy" href="/content-services/cic-overview/working-copy">
    Check out a document as an exclusive working copy to make edits without conflicting with other users.
  </Card>
  <Card title="Renditions" icon="images" href="/content-services/cic-overview/renditions">
    Automatically generate derived versions of documents — such as PDF previews or image thumbnails — via the Transformation Service.
  </Card>
  <Card title="Transformation Service" icon="wand-magic-sparkles" href="/content-services/cic-overview/transformation-service">
    A background service that converts documents between formats and indexes full-text content for search.
  </Card>
</CardGroup>

## Search Architecture

CIC's search capability is powered by **OpenSearch** (a managed Elasticsearch instance on AWS). HXQL queries can be executed via:

- The **Workspace** application's UI
- Direct calls to the **Query API**

<Tip>
  For full-text search, the Transformation Service must have processed the document to extract its text. Newly uploaded documents may take a short time before they are fully searchable.
</Tip>

### Example HXQL Queries

| Use Case | Example Query |
| --- | --- |
| Search by title | `SELECT * FROM SysFile WHERE sys_title LIKE '%Insurance%'` |
| Full-text search | `SELECT * FROM SysContent WHERE sys_fulltext = 'world'` |
| Filter by date | `SELECT * FROM SysFile WHERE sys_modifiedAt > '2024-01-01'` |
