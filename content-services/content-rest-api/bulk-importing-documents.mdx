Once you have acquired and used an access token (see Getting Started), you can start performing bulk import processes to ingest large volumes of documents into the repository efficiently and securely.

Note: This tutorial makes calls to Bulk Import API endpoints. For full reference information on all available Bulk Import API endpoints, see Bulk Import.
To perform a bulk import process:

1.  Open Command Prompt or Terminal.
2.  Enter a request similar to the following example to create a job for bulk importing documents into the repository:
```
curl -i -X POST [base_url]/api/import \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]" \
  -d "{\"name\": \"BulkImportJob1\", \"description\": \"MyPrimaryBulkImportJob\", \"creator\": \"john123doe456\", \"queue\": \"BulkImportQueue1\", \"uniqueKey\": \"bulk001import236\"}"
```

where the placeholders represent the following:

| Placeholder | Description |
| --- | --- |
| [base_url] | The URL of the Hyland Content repository |
| [access_token] | The access token you retrieved when you completed authentication |
| [product] | The name of the software you are using to make the request |
| [version] | The numbered version of the software |

A response that includes AWS credentials to access an S3 bucket as well as the jobId for the bulk import job is displayed, as shown in the following example:

```
{
  "awsAccessKeyId": "awsId1",
  "awsSecretAccessKey": "awsKey1",
  "awsSessionToken": "awsToken1",
  "bucket": "https://bulk-1234-sandbox-us-east-1-hy.s3.amazonaws.com/",
  "createdAt": "2025-05-20T04:21:32.943728813Z",
  "creator": "john123doe456",
  "description": "MyPrimaryBulkImportJob",
  "dryRun": false,
  "failedDocuments": 0,
  "importedDocuments": 0,
  "jobId": "c3814fe7-af63-4bd1-8906-d3c2453e091g",
  "name": "BulkImportJob1",
  "queue": "BulkImportQueue1",
  "status": "CREATED",
  "totalDocuments": 0,
  "uniqueKey": "bulk001import236"
}
```

3.  To upload your files and metadata to an S3 folder named after the jobId from the previous response, do the following:
1.  Prepare your files for upload by creating a JSON file named bulk-import-metadata.json, which contains the appropriate metadata for your upload files. Store this JSON file at the root level for the bulk import folder (equivalent to the jobId from the previous response).

2.  Using the awsAccessKeyId, awsSecretAccessKey, and awsSessionToken values from the previous response, enter a command similar to the following example in the AWS Command Line Interface (CLI):
```
export AWS_ACCESS_KEY_ID="awsId1"
export AWS_SECRET_ACCESS_KEY="awsKey1"
export AWS_SESSION_TOKEN="awsToken1"
```

3.  Copy contents to the S3 bucket by entering the following command in the AWS CLI:
```
aws s3 cp [source_location] [destination_location]
```

where the placeholders represent the following:

| Placeholder | Description |
| --- | --- |
| [source_location] | The path to the source location of the files you are uploading |
| [destination_location] | The bucket value from the previous response with the jobId value appended (for example, s3://bulk-1234-sandbox-us-east-1-hy/c3814fe7-af63-4bd1-8906-d3c2453e091g) |

4.  Start the bulk import job you created by entering a request similar to the following example (in Command Prompt or Terminal), using the same [jobId] value that was previously generated:
```
curl -i -X POST [base_url]/api/import/job/c3814fe7-af63-4bd1-8906-d3c2453e091g/start \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]"
```

The repository enqueues metadata files and workers process the job, loading metadata and content from the S3 bucket. The number of documents processed and any errors are tracked, and a response similar to the following example is displayed:

```

  "failedDocuments": 0,
  "importedDocuments": 0,

  "name": "BulkImportJob1",

  "startedAt": "2025-05-20T04:40:44.456264533Z",
  "status": "CREATED",
  "totalDocuments": 2,

}
```

5.  To check the status and other processing details for the bulk import job, enter a request similar to the following example, using the correct [jobId] for the job you are checking:
```
curl -i -X GET [base_url]/api/import/job/c3814fe7-af63-4bd1-8906-d3c2453e091g/status \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]"
```

6.  To see more details on failed documents for the bulk import job, generate a report for the job by entering a request similar to the following example:
```
curl -i -X GET [base_url]/api/import/job/c3814fe7-af63-4bd1-8906-d3c2453e091g/report \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]"
```

A response that includes a list of document IDs for any failed documents is displayed, as shown in the following example:

```
{
  "documentsFailed": [], //list of failed doc info, if any
  "job": {
    "bucket": "https://bulk-1234-sandbox-us-east-1-hy.s3.amazonaws.com/",
    ...
  }
}
```

8.  To abort a running bulk import job, enter a request similar to the following example:
```
curl -i -X POST [base_url]/api/import/job/fbd717cb-2813-40b9-a3a9-948a3e06916e/abort \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]"
```

A response that indicates a canceled status is displayed, as shown in the following example:

```
{
  "bucket": "https://bulk-5678-sandbox-us-east-1-hy.s3.amazonaws.com/",
  "createdAt": "2025-05-20T04:21:32.943728813Z",
  "creator": "john123doe456",
  "description": "MySecondaryBulkImportJob",
  "dryRun": false,
  "failedDocuments": 2,
  "importedDocuments": 20,
  "jobId": "fbd717cb-2813-40b9-a3a9-948a3e06916e",
  "name": "BulkImportJob2",
  "queue": "BulkImportQueue2",
  "startedAt": "2025-05-20T04:40:44.456264533Z",
  "status": "CANCELLED",
  "totalDocuments": 100,
  "uniqueKey": "bulk002import548"
}
```

9.  To refresh expired AWS credentials that are associated with a canceled bulk import job, enter a request similar to the following example:
```
curl -i -X POST [base_url]/api/import/job/fbd717cb-2813-40b9-a3a9-948a3e06916e/refresh-credentials \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]"
```

A response that includes the refreshed AWS credentials is displayed, as shown in the following example:

```
{
  "awsAccessKeyId": "awsId2",
  "awsSecretAccessKey": "awsKey2",
  "awsSessionToken": "awsToken2",
  "jobId": "fbd717cb-2813-40b9-a3a9-948a3e06916e"
}
```

10.  To delete a bulk import job, enter a request similar to the following example:
```
curl -i -X DELETE [base_url]/api/import/job/c3814fe7-af63-4bd1-8906-d3c2453e091g/purge \
  -H "Authorization: Bearer [access_token]" \
  -H "User-Agent: [product]/[version]"
```

All files, including versioned files, and metadata placed inside the S3 bucket are deleted.

Tip: Delete a bulk import job once you have finished ingesting the job's documents into the repository.
For more information on the response body elements shown in these examples, see Bulk Import.
