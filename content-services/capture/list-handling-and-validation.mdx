---
title: "List Handling And Validation"
---

While performing extraction verification of documents with a list field, you can type to search for a list item, but you must select a value from the items available in the drop-down list.

Consider the following validation rules for list fields:

- After a list value is specified and the batch is submitted, the list value remains unchanged, even if the list is later modified in the Administration interface.
- If a batch contains a list field that has already been indexed, but the list has been modified in the Administration interface in such a way that the indexed value is no longer available, a new value must be specified for the list field. You can select a new value only by reopening the batch.
- If a batch contains a list field that has already been indexed, but the field has been modified in the Administration interface in such a way that the list has been deleted, the list field reverts to a normal field and the previously indexed value remains as its specified value.
