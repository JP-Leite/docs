---
title: "Issue With Executing Commands And Streaming"
---

Issue: While trying to execute a command or perform streaming capabilities using Hyland Desktop Host, it fails to work as expected and the following error message is found in the DesktopHostCore_Log.json file: "Message":" chunk Size violated size limit exceeded".

Solution: Desktop Host is configured to exchange data and communicate with its underlying components through messages. You can configure the default message size for Desktop Host.

To configure the message size for Desktop Host:

1.  Open the config.json file in a plain-text editor with elevated privileges. In a typical installation, this file is in one of the following locations:
- After Windows installation, the file is in C:\ProgramData\Hyland\DesktopHost\bin.
- After macOS installation, the file is in /Library/Hyland/DesktopHost/bin.
2.  In the file, add the following text:
```
{
   "MaximumReceiveMessageSize": 128
}
```

Note: If the size of any message exceeds the default value of 128 KB, the connection will discontinue. Though a lower value may enhance performance, it is not recommended.
3.  Save the file and close the text editor.
4.  Restart the computer.
