Issue: While Desktop Host is started for the first time or a user logs on to a computer, Desktop Host faces an unexpected delay before starting.

Solution: Desktop Host is configured by default to load all services that are available in the DesktopHost\services directory. You can configure specific services present in the DesktopHost\services directory that are loaded when Desktop Host starts.

To configure Desktop Host to load specific services:

1.  Open the config.json file in a plain-text editor with elevated privileges. In a typical installation, this file is in one of the following locations:
- After Windows installation, the file is in C:\ProgramData\Hyland\DesktopHost\bin.
- After macOS installation, the file is in /Library/Hyland/DesktopHost/bin.
2.  In the file, add the following text:
```
{
   "WhitelistedServiceFolders": []
}
```

3.  For the key "WhitelistedServiceFolders", replace the default value [ ] with the name of the folders present directly under the DesktopHost\services directory and containing the required services.
A folder name must be included within double quotation marks, and multiple folder names must be separated with commas. For example:

```
{
   "WhitelistedServiceFolders": ["Example_Folder_1","Example_Folder_ 2"]
}
```

Note: If no value is provided for the key "WhitelistedServiceFolders", all services present in the DesktopHost\services directory are loaded. Ensure that each key value element is separated by a comma.
4.  Save the file and close the text editor.
5.  Restart the computer.
