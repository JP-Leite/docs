---
title: "Configuring Desktop Host To Load All Services"
---

Desktop Host is configured by default to load all services when the first connection is made by a client to Desktop Host. By default, the value for LoadServicesOnStartup is considered false to retain this existing behavior. If you need to load services on startup, you must add this configuration to the config.json file and set this to true. This section provides information on how to configure Desktop Host so that all services can be loaded when a user logs on to the system.

To configure Desktop Host to load all services:

1.  Open the config.json file in a plain-text editor with elevated privileges. In a typical installation, this file is in one of the following locations:
- After Windows installation, the file is in C:\ProgramData\Hyland\DesktopHost\bin.
- After macOS installation, the file is in /Library/Hyland/DesktopHost/bin.
2.  In the file, add the following text after an existing configuration:
```
"LoadServicesOnStartup": true
```

For example:

```
{
   "WhitelistedAddress": [
        "https://www.example1.com",
        "https://www.example2.com"
   ],
   "LoadServicesOnStartup": true
}
```

Note: Ensure that each key value element in config.json is separated by a comma.
3.  Save the file and close the text editor.
4.  Restart the computer.
