When indexing documents in the Metadata Panel, you must ensure all entered values are valid for their fields' metadata types before you can submit the batch for export. In some cases, entered values are automatically converted to the proper format for the field's type. However, if an entered value is flagged as invalid, an error message is displayed in red text below the field.

Note:
All metadata fields are limited to 1,024 characters. If metadata is exported from the Hyland Capture application and imported into a repository with more restrictive character limits, any metadata values exceeding the character limits set for the repository are truncated on import.
