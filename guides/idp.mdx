title: "Intelligent Document Processing (IDP)"
description: "AI-enabled document capture, classification and intelligent data extraction."
---

Hyland Intelligent Document Processing (IDP) provides AI-enabled document capture, classification, and intelligent data extraction to reliably improve efficiency, accuracy, and the speed of document processing. With its prompt-based configuration, you can quickly set up document processing workflows without extensive manual effort. It is deeply integrated into Automate and its orchestration engine to provide end-to-end automation at scale.

## Core Features

- AI-enabled document classification, separation, and field extraction
- Support for multiple document types and formats
- Configurable workflows for automated processing
- Integration with external repositories and business applications through Automate
- User-friendly interface for non-technical users
- Role-based access control for security and compliance

## Architecture Overview

<AccordionGroup>
  <Accordion title="Studio Modeler" icon="pen-ruler">
    Where professional developers and non-technical power users configure the IDP solution by leveraging AI and automation to streamline the extraction, classification, and processing of unstructured data and configure end to end business workflows that include conditional logic to route documents, trigger actions based on specific criteria, create, update, and release applications.
  </Accordion>
  <Accordion title="Studio Admin" icon="shield">
    Where administrators deploy applications, manage user permissions, and monitor deployed applications. A workflow engine for document processing that includes workflow steps like document ingestions, classification, separation, field extraction and document export.
  </Accordion>
  <Accordion title="Workspace" icon="table-columns">
    Where end-users can start new content-centric processes, and manage their work. A central and task-driven tool to enable human-in-the-loop interaction used for classification and extraction review.
  </Accordion>
</AccordionGroup>
