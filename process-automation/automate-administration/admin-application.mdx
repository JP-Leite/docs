---
title: "Admin Application"
---

The Admin Application is where released projects created in the Automate Modeler application can be deployed. Once a released project has been deployed it is referred to as an application.

The Admin Application is accessed using the /admin URL, for example:

https://hyland.com/admin

The following are the main areas the Admin Application covers:

- A DevOps section used to deploy projects, and manage and upgrade applications.
- A Process Admin section used to manage process instances, tasks and the audit details of all applications.
- A Process Admin section used to monitor behavior of the applications.
- An Application Capacity section used to show how many applications can be created and used at the same time.
