---
title: "Filtering Process Instances"
---

Process instances can be filtered by the following properties:

| Property | Description |
| --- | --- |
| Application name | The name of the application within which the process instance occurs. It is selected by default. |
| App version | The version of the application that the process instance is using. |
| Completed Date | Date of the process instance completion. |
| Id | The unique ID of the process instance. |
| Main process id | The unique ID of the main process. |
| Process Definition Name | The name given to the process as a part of modeling. |
| Process Name | The name given to the process instance when it was started. |
| Start Date | The time since the process instance was started. |
| Started By | The entity that started the process instance. |
| Status | The current status of the process instance. See the table below for a list of possible statuses. |
| Suspended Date | The date when the process instance was suspended. |

Note: Further details are available to view for each process instance by clicking on it. A properties panel will appear on the right-hand side of the screen.
The status of process instances are:

| Status | Description |
| --- | --- |
| RUNNING | The process instance is currently running. |
| COMPLETED | The process instance has been completed. |
| SUSPENDED | The process instance is currently suspended and cannot continue until it is reactivated. |
| CANCELLED | The process instance has been canceled and cannot be completed. |
