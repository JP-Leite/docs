You can filter the application instances in the Application Instances Released section, using the Filters section, in which you can add or remove filters as you need them.

To customize the filters used for the application instances list:

1.  In the Application Instances section, locate the Filters section.
2.  Use the Add filter button to add a filter that will be displayed on the project list.
3.  Use the x button next to the filter name to remove the filter from the project list.
4.  In the project list, click on the column name to make it the sorting column (ascending or descending). You can use the following filters:
| Property | Description |
| --- | --- |
| Application Name | The name of the application. |
| Project Release | The release of the application. |
| App version | The running version of the application instance. |
| Runtime version | Version of the Automate in which the application is running. |
| Modified | Date and time of the last update of the application. |
| Created | The time since the application was deployed. |
| Deployment Status | The status of the application. Once fully deployed it will display as Running. |
| Environment | Name of the environment where the application is running. |
| Workspace | A link to the user interface for the application. If there is more than one workspace, the default one set by the administrator in Studio Modeler is always displayed first on the list. |
