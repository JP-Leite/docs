The REST connector is used to provide a connection with a REST service. It can also be used to configure a webhook as an incoming trigger.

The REST connector appears on the process diagram as a pair of curly brackets.

The authentication used for the REST connector can be selected from the drop-down menu. All types of authentications are available. For more details, see Authentication.

Note: All REST services need to be separate from the production environment and are created and managed by customers.
The actions that can be executed using the REST connector are:

-  GET
-  HEAD
-  POST
-  PUT
-  PATCH
-  DELETE
-  OPTIONS
-  TRACE
