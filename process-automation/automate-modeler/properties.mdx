## Basic properties

The basic properties for a start event are:

| Property | Required | Description |
| --- | --- | --- |
| ID | Required | The unique identifier for the start event. This is system generated and cannot be altered, for example: StartEvent_1w29b3h |
| Name | Optional | The name of the start event. This will be displayed on the canvas. |
| Documentation | Optional | A free text description of what the start event does. |

## Redirect after submission

You can decide what happens after the form is submitted, using the Redirect after submission drop-down menu.

## Customize start form

You can use the Customise start form checkbox to adjust the following options:

-  Display start process button checkbox, which allows you to show or hide a start process button on the form and to change its name.
-  Display cancel button checkbox, which allows you to show or hide a cancel process button on the form and to change its name.

## Form name

An optional form can be used to begin a process. The form must exist within the same project as the process definition to be selected. Select a form from the drop-down list.

When a form is selected, an additional No login required checkbox is displayed. Check it to allow forms to be filled and submitted anonymously, without authentication.

## Mapping type

The mapping type sets how data is passed between the start event and the process. There are five options for sending this data.

If Map variables is selected for a process started via form submission, you can assign data from the existing form fields to the process variables, using the Input mapping and Output mapping sections.
