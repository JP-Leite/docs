---
title: "Inputs"
---

Inputs are the fields a decision table evaluates against. Process variables and a mapping type are used to pass the value of an input into the decision table to be evaluated.

You can add multiple inputs to the same decision table, using the + button in the header row.

Each input can have its Expression, Type, and Predefined value. The default type is string.

To change the input settings:

1.  Double click the Input column name.  Input settings are displayed.
2.  Enter the Expression that the input is going to use. Input entries use the FEEL (Friendly Enough Expression Language) language.
3.  Select the Type from the drop-down menu.
4.  (Optional) Add the Predefined values of the input.

You can add inputs as separate rows in the table and edit their values.Note:
Inputs can have a - in a column that matches any value passed in. This appears as blank in the associated XML.
