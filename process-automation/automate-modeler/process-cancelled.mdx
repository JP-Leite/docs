A process cancelled trigger is fired when a process is cancelled.

Specific details of when the trigger is fired. The configuration parameters are:

| Parameter | Definition |
| --- | --- |
| condition | A detailed condition for firing the trigger, created using the Condition Builder. For more details on conditions available as a part of the input values, see Creating Conditions. |
| name | Process instance name as given in theStudio Modeler application, for example My process name. |
| processDefinitionKey | Process definition key as configured in theStudio Modeler application. It has to be selected from a drop-down menu with a list of all available processes. |
| initiator | User name of the person who initiated the process. |
| businessKey | If exists, a business key associated to the process instance, for example Customer Review Process. |
| status | Status of the process instance, for example SUSPENDED. |
| cause | Cause of the process cancellation. |
