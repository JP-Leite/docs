---
title: "Creating And Editing User Task Notifications"
---

User task notifications are used to send email notifications during the process runtime. You can edit the default message to change the email that the assignee or the candidate receives when a task waits for their action.

To create user task notifications:

1.  Select Edit User Task Notifications against the correct process definition:
- if there is just one process definition, make sure no BPMN element is selected by clicking on a blank section of the canvas and the Edit User Task Notifications button will be visible in the right-hand properties panel.
- if the diagram contains more than one process definition, click on the individual pool to see the Edit User Task Notifications button.
The Edit User Task Notifications window is displayed.
2.  Select the tab in the top bar of the window to choose receiver of the email notification:
- select Assignee to send a notification to the user that is assigned to the task.
- select Candidate to send a notification to the user that is a candidate for the task.
Note:
You can use this window to send notifications to both the assignee and the candidate.

3.  Enter the email address that the notification is sent from in the From field.
4.  Enter the subject of the notification message in the Subject field. For example, The task ${taskName} awaits your response.
5.  Select the email template that you want to use under the Default email template drop-down menu. The available choices are:
| Email template option | Description |
| --- | --- |
| Default email template | The default email template used in the editor. |
| From file | One of the files uploaded to the application. You can see its contents in the Preview tab, and edit it in the Editor at the bottom. |
| From URL | URL address of the template. |
| Create new email template | Newly created email template specifically for this notification. You have to enter its name in the Template name field. You can see its content in the Preview tab, and edit it in the Editor tab at the bottom. |

When creating a new email template, you can use variables in the following format: ${variable}. See the table below for the description of the available varialbles.
| Variable | Description |
| --- | --- |
| taskUrl | URL address of the user task at hand. |
| taskId | Identifier of the user task at hand. |
| createdDate | Date of creation of the user task. |
| dueDate | Due date of the user task. |
| taskName | Name given to the user task. |
| firstName | First name of the user who receives the notification. |
| lastName | Last name of the user who receives the notification. |
| email | Email address of the user who receives the notification. |
| assignee | Name of the user who receives a candidate user task notification. |
| candidateUser | Name of the candidate user group to which the notification is sent. |

6.  Click Save to save the notification.

You are brought back to the process diagram.
