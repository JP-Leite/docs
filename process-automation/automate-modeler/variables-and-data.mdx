title: "Variables & Data Mapping"
---

Process variables are the primary mechanism for storing and transferring data between BPMN elements, forms, connectors, and scripts.

## Process Variables

### Scope and Creation
- **Scope**: Restricted to a single process definition (pool). Variables are not automatically shared between different pools in the same diagram.
- **Reserved Names**: Do not use `nrOfInstances`, `nrOfActiveInstances`, `nrOfCompletedInstances`, or `loopCounter`, as these are used internally by multi-instance elements.

### Variable Types
| Type | Example |
| --- | --- |
| **String** | `"Hello World"` |
| **Integer** | `642` |
| **Boolean** | `true` / `false` |
| **JSON** | `{"key": "value"}` |
| **Array** | `["A", "B", "C"]` |
| **Date/Datetime** | `2024-05-01` |
| **File / Folder** | References to objects in the repository. |

### Configuration Options
- **Ephemeral**: Use this for large payloads (e.g., API responses) that are needed during execution but shouldn't be persisted in the database to maintain performance.
- **Available in Web App**: Allows the variable to be displayed as a column in Workspace Task/Process lists.
- **Available in Analytics**: Enables filtering by this variable in the Studio Admin Dashboards.

## Data Mapping

Mapping defines how data moves between the process engine and external models (Forms, Connectors, Decision Tables).

### Mapping Strategies
- **Don't map variables**: No data transfer (Default).
- **Map All Variables**: Automatically maps inputs and outputs where names are identical.
- **Map Input Variables**: Only sends data *to* the model.
- **Map Output Variables**: Only receives data *from* the model.

### Explicit Mapping
Mapping is stored in the **Extensions Editor**. You can explicitly map a process variable to a model property, even if their names differ.
```json
"mappings": {
  "Task_ID": {
    "inputs": {
      "target_field": { "type": "variable", "value": "source_process_var" }
    }
  }
}
```

## Expressions & Hints

The **Expression Editor** allows you to use dynamic values and functions.
- **Helper Hints**: Hover over elements in the editor to see descriptions of available variables and event properties.
- **Functions**: Use built-in functions like `${now()}` for current timestamps or `${lookUp.property}` to reference nested JSON data.

### Declared Variables in Scripts
Within a Script Task, you can reference process variables using the `variables.` prefix:
```javascript
// Input from process
let cost = variables.unitPrice; 
let count = variables.quantity;

// Local calculation
let total = cost * count;

// Output back to process
variables.totalAmount = total;
```
