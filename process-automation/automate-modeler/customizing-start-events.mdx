The start event can be customized to display a non-standard start and cancel button labels. Instead of Start process or Cancel process, the button can be adjusted to the needs of the end user, reading for example: Submit form.

To set the customized event:

1.  Open the process.
2.  Select the Start event in the process.
3.  The Properties pane is displayed on the right. You can edit the following start event properties:

| Property | Description |
| --- | --- |
| Name | Customized name given to the start event. |
| Documentation | A link to your organization's process documentation. |
| Asynchronous Continuation | An option to execute the event in a background, using a separate transaction that runs concurrently to other transactions. |
| Redirect after submission | You can decide what happens after the form that starts the process is submitted, using the Redirect after submission drop-down menu. |
| Form name | You can select the form used to start the process from a drop-down menu. If no form is used, select None. |
| Redirection Options | You can edit or remove form buttons in this section. By selecting a checkbox next to the Start process or Cancel buttons, you add them to the form. You can edit their displayed names by filling in the Custom label fields. You can use the pencil button to open the Redirection Options dialog box, where additional buttons can be edited. You can define where the user is redicrected after hitting a particular button. |
| Mapping type | Select variable mapping type. For more information, see Mapping Process Variables. |
