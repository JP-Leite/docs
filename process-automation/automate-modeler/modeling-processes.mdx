---
title: "Modeling Processes"
---

The main purpose of the Studio Modeler application is to allow you to model processes for your own goals. You can use it to model the process diagram, edit any actions within the process, add and map variables and append any other elements defined in the application, such as forms or scripts.

By default, the new process is opened in the Diagram Editor. You can change the editor to edit the code, for further instructions see Choosing a Process Editor. When created, the default process diagram has only one Start Event (a thin circle), which is a starting point for your process. You can use it directly or delete it and change it to a new event.

You can add BPMN (see Adding Elements to Diagram) and engine events (see Engine Events) to your process, using the toolbox in the Diagram Editor. Then, when the events are added, you can map the process variables for these events, see Mapping Process Variables.
