---
title: "User Task"
---

User tasks represent a stage in the process where human action is required and cannot be omitted.

User tasks are displayed as a single thin, rounded rectangle with a user icon inside.

Human action is handled by a task being assigned to specific users or groups. The task that is assigned can be modeled using a form. After a task is completed, the process flow continues on to the next element in the process. When a new user task is added to a process diagram, it is automatically assigned to the process initiator.

## Task Assignment

User tasks can be assigned to a single user or to a group of candidates. To assign the tasks automatically, edit the Expression under the Assignment section of the Properties pane.

You can choose the task to be assigned to a static user, identity, or using an expression. When selecting from a pool of candidates, the task can be assigned manually or sequentially.

## Redirecting After Submission

When a user task is related to a form submission, you can decide what is going to happen after the form is submitted, using the Redirect after submission drop-down menu. The following options are available:
- Workspace
-
Landing page

You can set a customized Landing Page using a Rich text editor. It is available after the Landing Page option is set from the drop-down menu.

- URL (static)
- URL (query string)

## Task Due Date

The task due date can be set using one of the following options:
| Option | Description |
| --- | --- |
| Static date | A defined date set from a calendar. |
| Time duration | Period that has to pass for the task to be due. |
| Process variable | Value of a process variable that exists within the process. |

## Task Priority

The task can have a priority set for the purposes of filtering or assigning to the users.

## Form and Screen Options

You can select a form from a drop-down menu to be used as input for the user task.

If dedicated IDP screens exists, it is possible to select a Screen instead of a form. This allows the user who executes a task to use a dedicated user interface from IDP to complete the task.
