Errors are used by error catching events and error throwing events to model business exceptions using BPMN elements. They can be created and managed at the individual error event level, or at a diagram level. Unlike process variables, errors can be shared between process definitions in the same diagram.

To manage all errors in a diagram:

1.  Select Edit Errors against the correct process definition:
- if there is just one process definition, make sure no BPMN element is selected by clicking on a blank section of the canvas and the Edit Errors button will be visible in the right-hand properties panel.
- if the diagram contains more than one process definition, click on the individual pool to see the Edit Errors button.
The Process Errors window is displayed.
2.  To add a new error, click the + icon. The error is added as a new row.
3.  Click on the error to see its properties. You can change the Error name and Error code. The Error Id cannot be altered.
4.  To remove an error, click the trash can icon.
5.  Once editing is done, click Save.

You are brought back to the process diagram.
