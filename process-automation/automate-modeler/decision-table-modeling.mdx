Once created, the decision table has to be modeled to be used within the process.

You can use the Inputs and Outputs to create Rules that the table is going to manage. With Hit Policies, you can decide what behavior the table is going to exhibit, once the rule is filled.

When the table is complete, you can simulate its behavior.
