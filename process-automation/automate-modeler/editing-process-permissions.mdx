Process permissions are used to define who can start the process.

To define a new permission level:

1.  Select Edit Process Permissions against the correct process definition:
- if there is just one process definition, make sure no BPMN element is selected by clicking on a blank section of the canvas and the Edit Process Permissions button will be visible in the right-hand properties panel.
- if the diagram contains more than one process definition, click on the individual pool to see the Edit Process Permissions button.
The Process Permissions window is displayed.
2.  Select the Permission Level for the process. The available choices are:
| Permission Level | Description |
| --- | --- |
| Everyone can start it | Every user can start the process. |
| Nobody can start it | No user can start the process. |
| Specific users/groups | Specific users or groups of users can start the process. You must enter a value in at least one of the two fields: Candidate starter users or Candidate starter groups to define who can start the process. When you start typing, the application suggests users or groups for your search. |

3.  Click Save to save your choices.

You are brought back to the process diagram.
