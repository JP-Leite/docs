---
title: "Testing Agentic Task"
---

Once the Agentic Task is created, you can use the Test button to test it.

Contrary to other test runs, the Agentic Task is tested by deploying the agent into the environment with the updated configuration. This means that any changes made are implemented in the selected environment as soon as the agent is updated.

If the changes are not satisfactory, the changes have to be undone either by deploying the previous agent or process version, or by deploying the agent again with the previous configuration.
