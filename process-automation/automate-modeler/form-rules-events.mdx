---
title: "Form Rules Events"
---

Events are the specific moments that trigger execution of a form rule. Event sources can be form or field based.

The following form events are available:

| Event | Description |
| --- | --- |
| Form loaded | The form rule is triggered when the form is loaded. |
| Process finished | The form rule is triggered when a process is completed. You can use a correlationKey in the Payload section for this event to set a correlation key related to a defined starting form action. |

The following field events are available:

| Event | Description |
| --- | --- |
| Field value changes | The form rule is triggered when the field value is changed. |
| click | The form rule is triggered when the field is clicked. |
| blur | The form rule is triggered when the field value blurred. |
| change | The form rule is triggered when the field is changed. |
| focus | The form rule is triggered when there is focus on the field. |
| focusin | The form rule is triggered when there is focus inside the field. |
| focusout | The form rule is triggered when the focus moves outside of the field. |
| input | The form rule is triggered when value is filled in the field. |
| invalid | The form rule is triggered when the input value is invalid. |
| select | The form rule is triggered when the field is selected. |
