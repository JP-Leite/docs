---
title: "Decision Table Task"
---

Decision tables are used to manage business decisions within process workflows. They adhere to the Decision Model and Notation (DMN) standard. Decision tables take at least one input and have at least one output. The inputs are evaluated against a set of rules defined by the modeler and then produce the relevant output(s) that match those rules back to the process.

Decision table tasks are displayed as a single, thin rounded rectangle with a table icon inside.

Decision table tasks are essentially treated as service tasks and will always have the implementation value of dmn-connector.EXECTUTE_TABLE. The name of the decision table that is associated to the business rule task is used as the value under _activiti_dmn_table_ when viewing the Extensions Editor.
