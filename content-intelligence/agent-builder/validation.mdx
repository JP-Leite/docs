---
title: "Validation"
---

Validation of the deployed application is triggered automatically during the deployment.

The validation process is triggered every time one of the following happens:

- A project release is changed as a part of the Application Name step
- Environment is changed as a part of the Application Name step
- The Revalidate button is clicked during the Validation step

You are informed whether the validation was successful. If there are any errors during validation, they are displayed in the Validation dialog box.

The application can be deployed after the errors have been removed.
