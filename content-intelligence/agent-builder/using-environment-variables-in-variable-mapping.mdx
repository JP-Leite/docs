You can use the environment variables to call specific values and work with processes in different environments, such as staging vs production. After creating, you have to map the variables to be able to use them within your process or scripts. This example presents a script task variable, but a variable can be mapped using the Expression Editor whenever it is possible to add an expression to the variable value.

To map the environment variable:

1.  Create a script task. The task will be used to perform a script automatically.
2.  Create a script that can be called using a task from your process which requires the variable.
3.  From the Script Properties on the right, select Edit Script Variables.
4.  Select Create Script Variable.
5.  Enter an easily recognizable name. It will be used later to edit the variable.
6.  Select the Input category from the Variable category drop-down menu.
7.  Select the string type from the Type drop-down menu.
8.  Click the Save button to save the variable.
9.  Use the cloud button to save the script.
10.  Open the process and the task to which the script was assigned.
11.  Click the pencil button next to the newly created variable name.
12.  Select the Expression tab.
13.  Enter ${vars.[EnvironmentVariableName]}.   Note: The environment variable name will become available as auto-completion after the project is deployed.
14.  Save the variable.

The script task will use a different variable value, depending on the environment in which it is used.
