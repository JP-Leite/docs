## Managing Projects

## Projects

Projects are the top-level component of the business process being modeled. They contain all of the logic for the models that make up the business process such as forms, processes, content models, and connectors.

Some example projects are available for you to use to help you get started, see Using Example Projects.

## Saving Project Elements

To save a project element, select the cloud icon from the top bar.

If the element was updated, the icon is changed to black and can be used to save changes. If there are no updates, the icon is grayed-out.

## Deleting Project Elements

You can delete elements from your project.

To do it:

1.  Open the Agent Builder project.
2.  Find the element that you want to delete.
3.  From the top bar, select vertical ellipsis button >  Delete. A confirmation window is displayed.
4.  Click Confirm to delete the element.

The element is deleted.

## Searching Projects

You can search for any of the components that comprise your projects.

To search, click the magnifying icon located above the project tree. A search bar is displayed at the top of the interface, where you can enter your search terms. The search results are displayed in a list beneath the search bar and are dynamically updated as you type.

## Downloading Project Elements

You can download a project element as an XML file to edit it locally and upload it later. For more information on uploading project elements, see Uploading Project Elements.

To download a project element:

1.  Open the Agent Builder project.
2.  Find the element that you want to download.
3.  From the top bar, select vertical ellipsis button >  Download [element].

The file is downloaded.

## Duplicating Project Elements

Project elements can be duplicated and saved under a new name. To do this:
1.  Open the Agent Builder project.
2.  Find the element that you want to duplicate.
3.  From the top bar, select vertical ellipsis button >  Save as.
4.  Enter the new Name and an optional Description.
5.  Click Save.

The element is duplicated and added to the project. It can be seen in the left-hand pane.

## Validating Project Elements

Elements of the project can be validated on demand when created.

To validate a project element, open it and select the tick icon from the top bar. The validation is summarized in a message displayed at the bottom of the screen.

## Creating Projects

All actions within Agent Builder are done within a project. At least one project has to be created to start working.

To create a project:

1.  Sign in to Agent Builder. An All Projects window is displayed.
2.  Select how to create the project:
- To create a new, empty project, select Create Project.
- To upload an existing project .zip file into Agent Builder, select Upload Project.
3.  Enter a Project name and optional Project description if Create Project was chosen.  Tip:
You can use the Save as function to save a new copy of a project from the projects view screen or from the home screen. This is accessed by clicking the ellipsis and selecting Save as.

## Deploying Projects

Released projects can be deployed from the Project Releases menu in Agent Builder. Click the three dots adjacent to the project you want to deploy and select Deploy.

When deploying a released project, the user is presented with various configuration settings that may vary depending on the contents of the project:

-  Application Name
-  Validation
-  Permissions
-  Connectors

Select the Previous and Next buttons to navigate between the configuration pages. The current step is highlighted, and the completed steps are marked with a tick.

After completing the required configuration settings, click Deploy to close the configuration wizard and deploy the application.

## Downloading Projects

You can download the project as a zip file and see its content or edit it locally.

To do it:

1.  Sign in to Agent Builder. An All Projects window is displayed.
2.  Locate the project which you want to download. Select the vertical ellipsisbutton and click Download project name from the displayed menu. The project is downloaded as a zip file.

The downloaded zip file has a standardized folder and file structure, see Folder Structure and Files.

## Releasing Projects

Projects are version-controlled through the release action. When a project is first created, its version is set to 0 and increments by 1 every time it is released.

Note: A project must be released in order to deploy it for others to use. Version 0 of a project cannot be deployed.
To release a project:

1.  Save any changes to the project components.
2.  Find the Quick Run button and select the arrow down
3.  Select Release project.
4.  A dialog box with project data is displayed. Make sure it is correct, then select Confirm.

A confirmation message with the release version is shown.

## Deleting Multiple Projects At Once

If you have a role of a Modeler Administrator, you can delete multiple projects projects at the same time, using theEnable projects selection.

To delete multiple projects:

1.  Sign in toAgent Builder . An All Projects window is displayed.
2.  Select the Enable projects selection button at the right side of the window. Checkboxes next to the project names appear.
3.  Locate the projects that you want to delete. Select the vertical ellipsis button and select Delete selected projects from the displayed menu. You can also select all projects, using the Select all checkbox.
4.  In the displayed confirmation window, click Confirm. A confirmation window is displayed at the bottom of the page.

## Deleting Projects

You can delete the project when it is no longer needed.

CAUTION: This action cannot be undone.
To delete a project:

1.  Sign in to Agent Builder. An All Projects window is displayed.
2.  Locate the project that you want to delete. Select the vertical ellipsis button and select Delete project from the displayed menu.
3.  In the displayed confirmation window, click Confirm. A confirmation window is displayed at the bottom of the page.

Note: If you want to delete multiple projects, see Deleting Multiple Projects at Once.

## Downloading Project Releases

You can download a past release of the project to upload it to a different environment or work on it locally.

To do it:

1.  Select the project that you want to work on.
- Choose the project from the All projects drop-down list and select the three dots icon under the Options column.
- Open the project and click on its name in the top bar, then select View Project Releases.
2.  Click on the ellipsis button next to the release you would like to download in the Actions column.
3.  Select Download project release.

A zip file containing the project files from the selected release is downloaded.

## Duplicating Projects

It is possible to duplicate the project with all of its settings and properties and save it as a new one.

To do it:

1.  Sign in to Agent Builder. An All Projects window is displayed.
2.  Locate the project that you want to duplicate. Select the vertical ellipsisbutton and click Save as new project from the displayed menu. A Save as window is displayed.
3.  Enter the new Name.
4.  Click Save.

A new project is opened and ready to work.

## Editing Project Name

You can change the name and the description of the existing project.

To do this:

1.  Sign in to Agent Builder. An All Projects window is displayed.
2.  Locate the project the name of which you want to change. Select the vertical ellipsisbutton and click Edit project name from the displayed menu. An Edit Project window is displayed.
3.  Enter the new Project name and an optional Project description.
4.  Click Save. A confirmation window is displayed at the bottom of the page.

## Editing Project Release Names

You can change the name of a particular past release to easier distinguish between multiple releases.

To do it:

1.  Select the project that you want to work on.
- Choose the project from the All projects drop-down list and select the three dots icon under the Options column.
- Open the project and click on its name in the top bar, then select View Project Releases.
2.  Click on the ellipsis button next to the release the name of which you would like to change in the Actions column.
3.  Select Edit project release name. The Edit release window is displayed.
4.  Enter the new name of the release and an optional comment.
5.   Click Save. A confirmation message is displayed at the bottom.

The new release name is shown under Project Name column.

## Restoring Project Releases

You can restore any previous release of a project. In such cases, all project elements are replaced with those from the selected release.

Note:
It is recommended to save and release all changes you made in the project before you restore a release. For further instructions, see Releasing Projects.

To restore a previous release of a project:

1.  Select the project that you want to work on.
- Choose the project from the All projects drop-down list and select the three dots icon under the Options column.
- Open the project, click on its name in the top bar, and select View Project Releases.
2.  Click the ellipsis button next to the release you would like to restore in the Actions column.
3.  Select Restore Project to this Release. A confirmation message is displayed.
4.   Click Confirm.

The project is restored to the selected release.

## Uploading Project Elements

You can upload a project element as a JSON file and further edit it in the Agent Builder application. It can be especially useful if the element was downloaded and edited locally, or if it is added to a new application.

To upload an element:

1.  Open the Agent Builder project.
2.  To upload an existing element file into the application:.
- Select New > Upload >  and choose the type of element that you want to upload.
- Click the vertical ellipsis button next to the type of element that you want to upload in the left-hand pane and select Upload.
A Windows file selection window is opened.
3.  Select a JSON file from your drive and click Open. Wait until the file is uploaded.

The uploaded file is added to the project.

## Validating Projects

Validation is run when any model within a project is saved. Additionally, validation can be run on demand, using the Validate button at the top right corner.

The project level validation will be run against all models in the project.

Any validation errors in a model are displayed in the log at the bottom of the screen. You can expand the log by clicking on the bottom tool bar to view log details.

Validation errors include also errors related to features outside of the scope of the current subcription.

## Viewing Project Releases

Projects are version controlled through the release action. If a project was released multiple times, the history of its past releases can be seen any time.

To see the history of project releases:

1.  Sign in to Agent Builder. An All Projects window is displayed.
2.  Locate the project that you want to duplicate. Select the vertical ellipsis button and click View project releases from the displayed menu. A list of all past releases is displayed.

You can download a project release, restore a project release or edit a project release name from this window.
