The Application Name screen is the first one of the wizard helping you to deploy the application.

Tip: Projects are referred to as applications when they are deployed.
Enter the following details:

| Parameter | Description |
| --- | --- |
| Application Name | The name must be all lowercase and cannot exceed 26 characters. Alphanumeric characters and hyphens are allowed, though the name must begin with a letter and end with an alphanumeric character. For example: finance-application. |
| Project Release | A drop-down list with the available project releases that can be deployed. |
| Environment | A drop-down list with the available environments to which the application can be deployed. |
| Enable user task mail notifications | Once this checkbox is selected, a SMTP Configuration appears.  The drop-down menu is populated with Simple Mail Transfer Protocol (SMTP) authentications configured in Studio. If no SMTP configuration was added, only the Default SMTP Configuration option can be selected. This means that the mail notifications will be sent using the default Studio email address. If you want to use your own domain to send mail notifications, select the SMTP configuration for your domain. |
| Enable local development | Once this checkbox is selected, development of custom UI applications is possible. |

Once you are done editing this section, click Next.
