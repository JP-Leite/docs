When using more than one environment, for example a different one for staging and production, it is possible to set variables specific to each environment to avoid manually replacing environment-specific elements with each environment shift. These variables can be re-used between projects with different values to avoid creating multiple context-specific ones and improve variable maintenance. This can be especially useful when using a process in a staging environment first, and replacing the data in it when moving to production.

To create environment variables:

1.  Open the main Agent Builder page.
2.  Select Variables from the left pane. The Variables pane is displayed.
3.  Click the Create Variable button. The Create Variable dialog box is displayed.
4.  Enter the variable name into the Variable Name field. The variable name should be informative and easy to recognize. You will have to call the name later in the project configuration.
5.  Enter the value of the variable into the Value field.
6.  Select the environment from the Environment drop-down menu. If you select None, the variable is available in all environments globally, across the projects. If an environment is selected, the value is environment-specific.
7.  Select Create Variable.  The variable is created and assigned to the defined environment.

The created variable can be used in the expression editor to refer to a specific value. To apply different variable values to different environments, create another variable with the same name and assign it to a different environment.
