title: "IDP Project Management"
---

All document processing tasks in IDP are managed within a **Project**. Projects act as containers for your document classes, extraction prompts, and workflow configurations.

## Creating and Viewing Projects

### Creating a New Project
1. In the left navigation pane of Studio Modeler, click **All IDP projects** under IDP Configuration.
2. Click **Create Project**.
3. Enter a unique **Project Name** and an optional description.
4. Click **Create Project**. You will be redirected to the file selection page to begin configuration.

### Managing Existing Projects
From the **All IDP projects** list, you can view the status, creator, and last update time for every project. The following actions are available via the ellipsis (three dots) menu:
- **Rename/Edit**: Change the project name.
- **Delete**: Permanently remove a project.
- **Download**: Export the project configuration as a `.zip` file.
- **Collaborators**: Manage user access to the project.

## Project Configuration Settings

Project-level settings define the global behavior for document classification and extraction. These can be overridden at the individual class level if needed.

### Classification Thresholds
Access these via **Settings > General** in your project:
- **Class Review Threshold**: The minimum confidence required to classify a document without flagging it for manual review.
- **Class Assignment Threshold**: The absolute minimum confidence required for the system to assign a class to a document.
- **Class Distance**: The required confidence gap between the top match and the second-best match to ensure accurate classification.

## IDP Analytics & Reporting

IDP Reporting dashboards provide business analytics and performance metrics specifically for your document processing workflows. These insights help you measure automation ROI and identify areas for process improvement.

### Available Dashboards
- **Task Completion by User**: Tracks manual review productivity.
- **Automation Rate Summary**: Shows the percentage of documents processed without manual intervention.
- **Inbound/Outbound Volume Summary**: Monitors total document volume over time.
- **Classification Details**: Breakdown of document types and AI classification accuracy.
- **Current System Processing**: Real-time view of batches currently being handled.

### Accessing Reports
Analytics are managed within the **Studio Admin** environment:
1. Navigate to **Studio Admin**.
2. In the left pane, go to **Process Admin > Dashboards**.
3. Use the **Dashboards** filter to select your desired IDP report.
4. Click **Update** to refresh the visualizations.

## System & Service Limits

### Modelling Limits
- **Classes**: Max 350 per project.
- **Fields**: Max 100 per class.
- **File Uploads**: Max 100 files / 1000 pages / 100MB total during configuration.

### Processing & Service Limits
- **Batch Size**: Max 500 pages or 500 documents per request.
- **File Size**: Max 500MB per individual content file.
- **Page Limit**: Max 140 pages per document.
