---
title: "IDP Data Model: Classes & Fields"
---

The IDP Data Model defines how documents are categorised and what specific information is extracted from them. This is achieved through **Document Classes** and **Fields**.

## Document Classes

A document class groups documents with similar characteristics (e.g., Invoices, Contracts, IDs). Every document uploaded to IDP must be assigned to a class.

### Creating a Class
1. Navigate to **IDP Configuration > All IDP projects** and select your project.
2. Click **+ Create Class**.
3. Enter a unique **Name** and optional description.
4. **AI Suggestions**: If enabled, the system will suggest classes based on your uploaded documents. You can **Accept All**, **Reject All**, or manually refresh the list.

### Class Hierarchy (Parent & Child Classes)
- **Parent Class**: Acts as a template containing default fields. It is marked with a lock icon.
- **Child Class**: Inherits fields from the parent but can contain additional fields specific to its classified documents.
- **Detaching**: You can detach a child class from its parent. You must choose whether to **Remove Fields** linked to the parent or **Keep Fields** (converting them to individual fields).

---

## Class Properties & Settings

You can fine-tune class behavior via the **Options** tab:
- **Ignore Class for Auto-Classification**: Prevents the AI from automatically assigning documents to this class.
- **Use As Default Class**: Assigns all unclassified documents to this class to prevent them from being "stuck."
- **Detection**: Enable **Detect check boxes and radio buttons** for automated processing of UI elements.
- **Threshold Overrides**: Define confidence scores for **Field Review** and **OCR Review** at the class level to override project defaults.

---

## Field Extraction

Fields represent the specific data points you want to pull from a document (e.g., `Vendor Name`, `Total Amount`).

### Extraction Types
- **Text**: For standard textual data (names, titles).
- **Table**: For tabular data (line items, inventories). You must define columns for this type.
- **Reasoning**: Uses AI to analyze the document layout visually. OCR confidence is not tracked for this type.

### Creating and Configuring Fields
1. Select a class and click **Add Field**.
2. Define the **Field Name**, **Extraction Type**, and **Extraction Method** (Text vs. Full File).
3. **Full File Extraction**: Performs a deep analysis of the entire document rather than just visible text.

### Field-Level Settings
- **Formatting**: Configure case conversion, character removal (symbols/punctuation), and date reformatting.
- **Validation**:
  - **Internal**: Set Min/Max length, Regex patterns, or mark as "Required."
  - **Process-Based**: Trigger a script or existing workflow for complex validation.
- **Thresholds**: Set specific confidence thresholds for critical fields (e.g., 95% for Invoice Number) while allowing lower thresholds for less critical data.
