title: "IDP Forms & Validation"

IDP allows you to customize the user interface for document review and establish rigorous validation rules to ensure data quality.

## Custom Verification Forms

Custom forms allow you to arrange fields and configure external system lookups for manual document review.

### Form Generation
- **Automatic Creation**: When you add at least one field to a class in IDP Configuration, a corresponding form model is automatically generated.
- **Inheritance**: For child classes, form fields are inherited from the parent. If you detach a class, you can choose to keep or remove these inherited form fields.
- **Synchronisation**:
  - **Adding/Updating**: Adding or renaming a field in the IDP config automatically updates the form.
  - **Formatting/Validation**: Rules set in the IDP config are reflected in the form. It is best practice to manage these rules in the configuration UI rather than the form builder.

To customize a form, select a class in Modeler and click **Navigate to Form**.

---

## Document Upload Forms

Before a process can start, a form is needed to ingest documents.
1. In Studio Modeler, select **Create > Form**.
2. Name it "Upload Form" and drag the **File Upload** element onto the canvas.
3. **Advanced Settings**: Enable **Allow multiple file attachments** to process batches.
4. Link this form to the **Start Event** of your IDP workflow.

## Field Validation & Formatting

Validation rules act as a "gatekeeper," preventing documents with errors from moving forward until they are reviewed.

### Validation Types
- **Internal Validation**:
  - **Field Length**: Set min/max character limits.
  - **Regex**: Ensure values match specific patterns (e.g., email or tax ID).
  - **Required**: Mark mandatory fields.
- **Process-Based Validation**: Use a **Custom Validation Script** in Modeler for complex logic, such as performing a database lookup or calling an external API.

### Output Formatting
Standardize your extracted data before it reaches downstream systems:
- **Case Conversion**: Automatically switch to UPPERCASE, lowercase, or Sentence case.
- **Character Removal**: Strip out punctuation or symbols.
- **Date Reformatting**: Force all dates into a consistent format (e.g., `DD-MM-YYYY`).
- **Truncation**: Trim values to a specific character length.

---

## Custom Validation Scripts

IDP provides templates to help you build complex validation logic using scripts.
1. In the **Edit Field** pane, click **Add** under Field Validation.
2. Select **Create Process**.
3. Use the provided **Script Template** to define your logic.
4. The system will save the process under **Processes** and the corresponding script under **Field Validations**.
