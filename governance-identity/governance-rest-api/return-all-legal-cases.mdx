Returns a map of all legal cases in the database.

## Base Request

```
GET /api/legal-cases
```

## Query Parameters

| Parameter | Description | Type | Required | Notes |
| --- | --- | --- | --- | --- |
| limit | The maximum number of legal cases to evaluate when searching the database. | integer | Optional | Default is 10. |
| exclusiveStartKey | The key of the last legal case on the previous page of search results. | string | Optional | Can be used in conjunction with the limit parameter to implement pagination of Governance search results in the Workspace application. |
| ascendingOrder | True/false indicator of whether to list the retrieved legal cases in ascending order. | Boolean | Optional | Default is false. |

## Headers

[Mozilla developer documentation](https://developer.mozilla.org/)

| Header Name | Description | Required | Values |
| --- | --- | --- | --- |
| Authorization | The access token | Required | Bearer [access_token] where [access_token] is a string generated when you complete authentication (see Getting Started) |
| User-Agent | Identity of the client making the request to the server | Required | [product]/[version] where [product] is the name of the software you are using to make the request and [version] is the numbered version of the software (for example, MyCompanyProduct/1.0). For information on other types of values commonly included in user-agent request headers, see the User-Agent HTTP header reference section on the  website. |
| Accept | The format of the data to be returned | Optional | application/json |

## Sample Request

```
GET /api/legal-cases?limit=20

Authorization: Bearer [access_token]
User-Agent: MyCompanyProduct/1.0
Accept: application/json
```

## Response Body

| Element | Description | Type | Notes |
| --- | --- | --- | --- |
| contents | List of content objects for the legal case. | array | None |
| lastEvaluatedKey | The key of the last evaluated legal case in the search results. | string | None |

See Legal Case for more information on the additional, more common elements listed in the body of the following sample response.

## Status Codes and Errors

The following table lists the returned status codes.

| Code | Description | Notes |
| --- | --- | --- |
| 200 | OK | Returns a map of all legal cases in the database. |
| 401 | Not Authorized | Authentication has failed or was not provided. |
| 403 | Not Allowed | Access permissions are invalid or missing. |
