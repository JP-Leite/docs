---
title: "Creating A Retention Policy"
---

To deploy a Governance project, each category should have a retention policy. If there is no retention policy set for a category, you will not be able to deploy a file plan. Before creating a retention policy make sure there is a proper content type created for the project. For mor information see Content Type Configuration.

To create a retention policy:
1.  Open your project in Studio Modeling.  Note: If the file plan in your project is already opened, go to the point 3.
1.  Go to All Governance projects under  Governance Configuration.
2.  Select your project. The Governance project site is displayed.
2.  Within the file plan open the category where you want to create a retention policy. The Retention section is displayed on the right side of the screen.
3.  Set the Cutoff to select a retention trigger:
-  Immediate: The retention starts immediately after the record is assigned to the category.
-  Property-based date: The retention start is based on the date of a defined property, such as creation date or modification date. You can use a system property date or create a custom property date.
Note: To use a custom property based date for a retention policy, the property has to be created prior to the project deployment. For information on creating a custom property date, see Creating a Property Date. For an information on property-based date options, see Record Lifecycle Flow
4.  Click Save to save the cutoff for the category.
5.  Set the Retention Period to define how long the document should be retained:
-  Retain content until retention adjusted: The record is kept until the property date is reached. If an extra retention time after trigger is set up, the disposition date is calculated accordingly. To enable the selection of the retention adjustment trigger, select Apply Retention Adjustment and choose a property date. If you want to keep the record for additional period after the property date is reached select a number and a time unit in Add extra retention time after trigger.
-  Retain content for a fixed period of time: The record is kept for a fixed period that is defined by the user. The calculated disposition date is displayed in the Record Management tab in Governance.
6.  Click Save to save the retention period for the category.
7.  Set the Disposition to define what should happen to the record after the retention period comes to an end. Available options:
-  Ready for Purge: The record can be deleted after the retention period is over. This action must be completed by the Record Manager manually. For more information on deleting the records, see Disposing of Records Past Retention.
-  Workflow The record is deleted automatically when it reaches Reached Disposition status. The process of the removal is triggered once a day and does not require approval to remove the records. When the records are removed, the removal operation is logged for audit purposes.Note: This is an out‑of‑the‑box workflow added in the Process tab in Studio Modeler. Do not change this process. This workflow can be used in multiple categories.
8.  Click Save to save the disposition for the category.
