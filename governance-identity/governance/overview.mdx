This section guides you through creating a Governance project, setting a retention policy, and deploying the project so that records management is enforced across your CIC repositories.

## Governance Workflow

<Steps>
  <Step title="Create a Governance Project">
    Create and configure a Governance project in Studio Modeler. This is the container for your file plan and retention configuration.

    [Create a Governance Project →](/governance-identity/governance/creating-a-governance-project)
  </Step>
  <Step title="Create a File Plan">
    Define a file plan that sets retention and disposition schedules, ensuring records are maintained for the correct duration according to regulations.

    [Create a File Plan →](/governance-identity/governance/creating-a-file-plan)
  </Step>
  <Step title="Create a Category">
    Add a category to your file plan to group related records and apply retention rules at the right granularity.

    [Create a Category →](/governance-identity/governance/creating-a-category)
  </Step>
  <Step title="Set a Retention Policy">
    Create a retention policy for the category to define exactly how long records are kept and what happens when they expire.

    [Create a Retention Policy →](/governance-identity/governance/creating-a-retention-policy)
  </Step>
  <Step title="Deploy the Project">
    Release and deploy the Governance project to make the configuration active. Once deployed, users can manage records in Workspace.

    [Release a Project →](/governance-identity/governance/releasing-a-project) · [Deploy a Project →](/governance-identity/governance/deploying-a-project)
  </Step>
</Steps>
