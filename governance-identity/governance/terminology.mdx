This topic defines key terms related to Content Innovation Cloud Governance.

##  Governance

Hyland Governance within Content Innovation Cloud supports content retention and record management. A Governance project is configured in Studio Modeler. The Governance App allows you to create a retention policy using a predefined content type. A content classification in Governance is focused on a content type. When the content type is added to the retention policy all documents classified as this content type is subject to this retention policy. Records representing content in the Hyland Cloud Content repository are created and can be managed in Workspace.

## Content Type

A content type is the set of properties that define a content object. Each content type encompasses metadata fields that are associated with its content items. Metadata ensures that relevant information is captured and organized systematically. The properties and behaviors for a content type are defined through schemas and mixins. There are default content types with the default metadata and properties. However, you can also compose your own content type of custom-made schemas and mixins. When assigned to the proper category for a retention policy, all documents with this content type are subject to the rules set in the Governance project configuration. For more information on creating a content type, see the Creating Content Types section in the Hyland Automate Modeler documentation.

## CIC Repository

The repository stores and serves the documents within the Content Innovation Cloud. Similar to a database, the repository embeds logic to ensure the security and availability of documents. The repository also organizes the documents in a hierarchy. For more information on the  Content Innovation Cloud  Repository, see the  Content Innovation Cloud Overview documentation.
