---
title: "Creating A Category"
---

A category allows for a content classification that contains a retention policy for records. It controls how records are managed and how they are disposed of when they are no longer needed. Each category can only have one retention policy.

To create a category:

1.  Open your project in Studio Modeler. To select a project:
1.  Go to All Governance projects under  Governance Configuration.
2.  Select your project. The Governance project site containing File Plan Categories is displayed.
2.  Click the Create Category button in the upper right corner. The Create Category window is displayed.
3.  Enter a unique category name in the Category Name field.  Note: You cannot enter a name that is already used by another category in the selected project. It is possible to have the same name on two different projects.
4.  Click OK. A side panel containing Overview and Retention sections is displayed on the right side of the screen.
5.  Select a Record Type to set a content type that will be used for the category. The Record Type section contains a dropdown listing Governable Content Types in the selected Governance Project.   Note: For more information on content types, see Content Types in the Hyland Automate Modeling documentation and Terminology.   A new category is created and the Governance project site is displayed. The category is added to the File Plan Categories.
6.  Set the retention policy for the category. To create a retention policy, see Creating a Retention Policy.
If the category does not have a retention policy, it is labeled in the categories list with a status Not configured.

7.  Click Save. The new category is created, and the list of categories is displayed. Under the Category Name  you can see the path pointing where the category is placed.
