An upgrade option allows you to update the changes made within the Governance project, such as changing a file plan configuration, adding a new category, or changing retention policy.

If your Governance project is deployed and you made changes within this project for the changes to be applied you need to use the Upgrade option in Studio Admin. The retention rules changes are applied only to the new records. They do not affect existing records.Note: The upgrade does not support deleting categories and Content Focus changes when the category is associated to at least 1 record.
To upgrade a Governance project:

1.  Unfold the Devops tab in Studio Admin.
2.  Click Application Instances. and find your Governance project.
3.  Click the three dots next to the Governance project you want to upgrade.
4.  Click Upgrade. An Upgrade dialog box is displayed.
5.  Click Upgrade. Prior to upgrading you can verify Application Name details, Variables, Validation, Permissions, and Runtime Version. A message confirming the application upgrade is displayed.
