Governance integrates with Hyland Experience Repository (HxPR) to manage records. Below you can find information on when records enter Governance and what triggers further status changes.

A record enters retention after it is triggered. Triggers can occur immediately or based on a specific date. They can be based on the system's current date, the date the item was created, the date the item was last changed, or a custom date you set manually:

| Trigger Type | Description |
| --- | --- |
| Immediate | The retention starts as soon as the document is uploaded. |
| createdDate | The retention starts from the date the document was created. If the record is updated later, the retention period continues to count from the original creation date. This is a system property. |
| modifiedDate | The retention starts from the last time the document was changed. This property is used for documents that evolve over time and need to be retained for a set period after their most recent updates. This is a system property. |
| Custom Date Property | The retention starts on the date specified in this property. The custom date property field is created in Studio Modeler prior to creating a retention policy. The value for this property is set later in HxPR when HR or Record Manager updates the record. |

Once retention is triggered, the system sets a retention start date and associates the records with its governing policy. During the retention period, the record is preserved and protected against unauthorized alteration or deletion until disposition actions are executed in accordance with the schedule.

Mutual communication between Governance and HxPR allows for discovering new or updated records and bringing metadata from Governance to the repository and conversely. If the property with a cutoff date is used in the repository then this cutoff date is assigned in Governance. Governance creates a record and the system runs periodically to check and update record status. When the cutoff date matches the current date, the record is marked Under Retention.

The following sequence shows the lifecycle of records:

Ready - When the file upload is completed in HxPR and the cutoff date is in the future, the file is waiting until this date is reached and the record status is Ready. When a record reaches the cutoff date, the status changes to Under Retention.

Under Retention - When a file is uploaded in the repository and reaches a cutoff date, then it enters retention. The records are processed by the system at intervals. If a file's cutoff date is in the past, it enters retention immediately. Records have the Under Retention status until the disposition date is reached. The disposition date is calculated from the cutoff date and the retention policy's duration.

Reached Disposition - The retention duration can be infinite or set for a fixed period of time. When the disposition date is reached, the system moves the file to the Reached Disposition status.
