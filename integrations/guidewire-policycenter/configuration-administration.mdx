## Adding Default User Groups Within Sso Identity Provider

When using a corporate identity provider (e.g., Okta) for Single Sign-On, an administrator may manage default user groups directly within the identity provider server. This topic describes how to ensure that CIC user group assignments synchronize with the corporate identity provider.

To synchronize CIC user group assignments with the corporate identity provider:

1.  Within the corporate identity provider, create the following user groups. For more information on creating user groups, refer to your corporate identity provider's documentation.
- Guidewire PC Content Users
- Guidewire PC Doc Security - None
- Guidewire PC Doc Security - Sensitive
- Guidewire PC Doc Security - Unrestricted
- Guidewire PC Doc Security - Internal Only
2.  Within the corporate identity provider, add all applicable users to the new user groups.
3.  Log in to the the CIC Administration Portal and ensure that the corresponding user groups with the same name are marked as External. For more information, see the Content Innovation Cloud Administration Portal documentation. The next time a user logs in, their user group assignment within CIC will be synchronized with what is configured within the corporate identity provider.Tip: It is recommended to set the Guidewire PC Solution Administrators user groups to Internal and administer that user group exclusively within the CIC Administration Portal.

## Configuring An Administrator User For Single Sign On Sso

You must configure an administrator user within PolicyCenter to work with single sign-on (SSO). See your Guidewire PolicyCenter documentation for detailed information on configuring users.

PolicyCenter must be configured and deployed in the Guidewire cloud environment with a single sign-on (SSO) identity provider.
To configure an administrator user for single sign-on (SSO):

1.  Log in to PolicyCenter as an unrestricted user before enabling single sign-on (SSO).
2.  If desired, create a new user that will serve as the single sign-on (SSO) administrator. The unrestricted user can also be used for this purpose if preferred.
3.  Grant your chosen user single sign-on (SSO) administrative rights. In most cases the user will need to be granted the All Permissions role and the System Administrators group. For information on giving a PolicyCenter user single sign-on (SSO) administrative rights, see your Guidewire PolicyCenter documentation.
4.  Log in to the Single Sign-On (SSO) Identify Provider Server as an administrator.
5.  Create an account with the same user name as your PolicyCenter SSO administrator.
6.  Assign the Single Sign-On (SSO) administrator user to the appropriate SSO group.
7.  Enable Single Sign-On (SSO) for that user and save all changes.

## Configuring Credentials In Policycenter

Once the service user is created and configured, the credentials must be saved for use in PolicyCenter.

To configure credentials in PolicyCenter, you must create a new secret with information from your CIC service user. See your Guidewire PolicyCenter documentation for information on creating a secret in Guidewire.

When you add your secret in PolicyCenter, include the following information:

- Set the key to uceb.api.auth
- Set the value to the Client ID and Client Secret of the CIC service user created for use with your PolicyCenter integration. Use a colon to separate the two values. For example, if the service user's Client ID is abc12345 and the service user's Client Secret is xyz67890, the value should be set to abc12345:xyz67890

## Configuring External System Settings

You must configure the external system settings with the appropriate scopes, environment, and application for your solution.

To configure the external system settings:

1.  Log in to the CIC Administration Portal using the assigned administrator account of your CIC instance.
2.  Click the External Systems tab in the top menu.
3.  Click External Applications in the left pane. The External Applications page is displayed.
4.  Click on the external application that was used when the service user was created. For example: PolicyCenter_Backend_Client. See Creating a Service User for information on creating a service user. The application's page is displayed.
5.  Click the Allowed Scopes drop-down list and select all desired scopes for your solution. The following scopes are required:
- environment_authorization
- hxp.nucleus.account
- uceb
6.  If you have an Environment-level application, an Environment drop-down is displayed below the Allowed Scopes drop-down list. Click on the drop-down list and select the desired environment.
7.  If you have an Account-level application, an Application drop-down is displayed below the Allowed Scopes drop-down list. Click on the drop-down list and select Hyland for Guidewire PolicyCenter.
8.  Click Save.
9.  Click Close. You are returned to the External Applications page.

## Configuring Guidewire Home

You must configure the planets in your Guidewire Home application for OAuth.

To configure OAuth in Guidewire Home:

1.  In Guidewire Home, navigate to Variables.
2.  Add a new variable for each planet that you want to enable OAuth for.
3.  Set the Application to Policy Center.
4.  Set the Key to PLUGIN_AUTHENTICATIONVERIFIER_SUBJECTMAPPINGS_`<sub>`, with `<sub>` replaced with the Client ID you were provided by Guidewire.
5.  Set the Value to the user name of the service user you want the CIC Document Upload feature to use.  Note: When the CIC Document Upload feature performs an action in Guidewire, it will be logged under this user. It is recommended that a dedicated service user is used.
6.  In Guidewire Home, navigate to Planets.
7.  Select your planet and navigate to Insurance Suite.
8.  Select variables.
9.  Navigate to the section for Policy Center and locate the DEPLOYMENT_ID variable.  Note: If the DEPLOYMENT_ID variable does not exist, you will have to add it using Guidewire Resource Notation (GRN). Refer to Guidewire's documentation for details on adding variables using GRN.
10.  Restart the affected planets.

## Configuring New Users For Single Sign On Sso

After configuring an administrator user for SSO, you must configure all other user accounts you wish to use SSO. See your Guidewire PolicyCenter documentation for detailed information on configuring users.

To configure new users for SSO:

1.  Log in to PolicyCenter as the administrator user configured for SSO. See Configuring the Single Sign-On Identity Provider Server for information on configuring an administrator user for SSO.
2.  Create a new user with the necessary information, desired role, and group. For information on creating new users in PolicyCenter, see your Guidewire PolicyCenter documentation.   Note: Ensure that the user name for each new user is in the format of an email address with the correct email domain enabled for Single Sign-On (SSO).
3.  Log in to the SSO Identify Provider as an administrator.
4.  Create a new account with the same user name in your SSO Identify Provider. For information on creating new users in your SSO Identify Provider, see your SSO Identity Provider's documentation.
5.  Assign each of the new users created in step 2 to the appropriate SSO group. Enable SSO for those users and save all changes.

## Configuring The Guid

You must configure theGuidewire PolicyCenter Workspace project so that it contains the GUIDs for the Hyland Content Innovation Cloud’s security user groups.

You must have:
-
Administrator access to Guidewire PolicyCenter

-
Administrator access to the Hyland Content Innovation Cloud.

To update PolicyCenter with the required GUIDs:

1.  Log in to the CIC Administration Portal.
2.  Navigate to Identity > User Groups. The User Groups pane is displayed.
3.  Click the Guidewire Doc Security - None user group.
4.  Copy the GUID for the user group, which is displayed at the end of the URL in the address bar. For example, if the URL in the address bar is https://test.hyland.com/identity/user-groups/12345678-abcd-1234-abcde-12345789abc, the GUID you should copy is 12345678-abcd-1234-abcde-12345789abc.
5.  Update the DocSecurityType_null runtime property with the copied GUID. See Configuring PolicyCenter Runtime Properties for more information.
6.  Repeat these steps for the other security types (Guidewire PC Doc Security - Sensitive, Guidewire PC Doc Security - Unrestricted, and Guidewire PC Doc Security - Internal only), matching the name of the user group with the key in the mappings file.
7.  Open the Studio Modeler application in the Hyland Content Innovation Cloud.
8.  Click on the Guidewire PolicyCenter Workspace project. The project window is displayed.
9.  Click the Upload doc to Guidewire CC process in the Process Automation pane on the left. The project page is displayed.
10.  Click Edit Process Variables in the Properties pane on the right. The Process variables page is displayed.
11.  Locate the docSecurityGuidNone variable and click the corresponding Edit variable button. The Edit variable page is displayed.
12.  In the Default value section, enter the GUID for the user group copied in step 4 in the Provide a value field.
13.  Click Save. The Process variables page is displayed.
14.  Locate the docSecurityGuidSensitive variable and click the corresponding Edit variable button. The Edit variable page is displayed.
15.  In the Default value section, enter the GUID for the user group copied in step 4 in the Provide a value field.
16.  Click Save. The Process variables page is displayed.
17.  Locate the docSecurityGuidUnrestricted variable and click the corresponding Edit variable button. The Edit variable page is displayed.
18.  In the Default value section, enter the GUID for the user group copied in step 4 in the Provide a value field.
19.  Locate the docSecurityGuidInternal variable and click the corresponding Edit variable button. The Edit variable page is displayed.

## Configuring User Groups

When a new environment is provisioned, several user groups are created by default. Do not delete or edit the default user groups unless directed to by a Hyland Administrator. If you need to create a new user group, see the Content Innovation Cloud Administration Portal documentation. Users must be manually added to these user groups. For general information on assigning users to HxPR user groups, see the Content Innovation Cloud Administration Portal documentation.

The following user groups exist by default:

| User Group Name | Description |
| --- | --- |
| Account Administrators | This user group provides configuration rights in the CIC Administration Portal allowing for general platform administration. |
| Guidewire PC Content Users | This user group allows users to upload documents both from within Guidewire PolicyCenter and from within the Hyland CIC Platform. Note: Access to content may be further restricted by Document Security configuration settings. For more information on Document Security, refer to Configuring Document Security. |
| Guidewire PC Solution Administrators | This user group provides configuration rights for backend processes such as document upload. |
| Guidewire Doc Security - None | This user group corresponds to the PolicyCenter Security Type that is a blank field. PolicyCenter users who are a part of this group can view documents that have been uploaded with the Security Type none. If no specific permissions are set to the Unrestricted Security Type in Guidewire PolicyCenter, users who are assigned to the Guidewire Doc Security - None user group will also be able to view documents uploaded under the Unrestricted security type. CIC users assigned to the Guidewire Doc Security - None user group can only view documents uploaded under the Security Type none. When a document is uploaded to the Guidewire Doc Security - None Security Group, the guid of this user group is added to the Document Security Type ID property for the document. |
| Guidewire Doc Security - Sensitive | This user group corresponds to the Sensitive Security Type set in Guidewire PolicyCenter. PolicyCenter users assigned to this user group can view documents uploaded to the None, Unrestricted and Sensitive Security Types. CIC users assigned to the Guidewire Doc Security - Sensitive user group can only view documents uploaded under the Security Type Sensitive |
| Guidewire Doc Security - Unrestricted | This user group corresponds to the Unrestricted Document Security Type in Guidewire PolicyCenter. PolicyCenter users assigned to this user group can view documents uploaded under the Unrestricted Document and None Security Types.CIC users assigned to the Guidewire Doc Security - Unrestricted user group can only view documents uploaded under the Security Type Unrestricted.  When a document is uploaded to the Guidewire Doc Security - Unrestricted Security Group, the guid of this user group is added to the Document Security Type ID property for the document. |
| Guidewire Doc Security - Internal only | This user group corresponds to the Internal only Document Security Type in Guidewire PolicyCenter. PolicyCenter users assigned to this user group can view documents uploaded under the Internal only Security Type.CIC users assigned to the Guidewire Doc Security - Internal only user group can view documents uploaded under the Security Type Internal only.  When a document is uploaded to the Guidewire Doc Security - Internal only Security Group, the guid of this user group is added to the Document Security Type ID property for the document. |

## Creating A Service User

You must create a service user to facilitate communication between Guidewire PolicyCenter and the Hyland CIC Platform.

Before creating the service user, appropriate user groups must be configured. For more information on configuring user groups, see Configuring User Groups.
To create the service user:

1.  Log in to the CIC Administration Portal using the assigned administrator account of your CIC instance.
2.  Click the Identity tab in the top menu. The Users  page is displayed.
3.  In the left pane, expand the Users drop-down and click Service Users. The Users page is filtered to only display service users.
4.  Click Create Service User. The Create Service User wizard is displayed.
5.  Enter a user name in the Username field, then click Next Step. The User Groups (optional) page is displayed.
6.  Move all desired user groups to the Assigned field by clicking on them. The following groups are required:
- Guidewire PC Content Users
- Guidewire PC Doc Security - None
- Guidewire PC Doc Security - Sensitive
- Guidewire PC Doc Security - Unrestricted
- Guidewire PC Doc Security - Internal only
7.  Click Next Step. The Register New Client (Optional) page is displayed.
8.  In the Application Name field, enter the name of the external system. For example: PolicyCenter_Backend_Client.
9.  Click Next Step. The Review page is displayed.
10.  Review your changes, then click Create Service User. The Summary page is displayed.
11.  Copy the Client Id and the Client Secret into a secure password store.  CAUTION: The Client Secret will not be displayed again once this page is closed. If you lose the Client Secret, you will need to reset the Client Secret in the CIC Administration Portal and change its stored value in PolicyCenter.
12.  Click Close. The service account is created and added to the Users list.

## Creating An Identity Provider In Cic

You must create an identity provider within the CIC Administration Portal which corresponds to the Single Sign-On (SSO) Identity Provider Server that will be used to authenticate in Guidewire PolicyCenter.

To create an identity provider within the CIC Administration Portal:

1.  Log in to the CIC Administration Portal.
2.  Navigate to Identity > Identity Providers. The Identity Providers page is displayed.
3.  Click Add Identity Provider. The Add Identity Provider wizard is displayed.
4.  Type the name for the Identity Provider Client in the Identity Provider Name field.
5.  Type the entityID for your Identity Provider Server in the Entity Id field. See the Identity Provider Server documentation for details on finding this value.
6.  Type the metadata URL for your Identity Provider Server in the Metadata URL field. See the Identity Provider Server documentation for details on finding this value.
7.  Select RSA-SHA256 from the Minimum Assertion Signature Algorithm drop-down.
8.  Click Next Step. The Bindings page is displayed.
9.  Enable the Override authentication request binding option and select Redirect from the first Binding Type drop-down.
10.  Enable the Override assertion binding option and select Post from the second Binding Type drop-down.
11.  Click Next Step. The Claim Type Mapping page is displayed.
12.  Enter the required details for the identity provider you are adding. Ensure that the claim names match in the Service Provider Application of the selected Identity Provider Server.
13.  Click Next Step. The Email Domains page is displayed.
14.  Enable the email domain(s) corresponding to the Single Sign-On (SSO) Identity Provider Server that will be used to authenticate in Guidewire PolicyCenter. If the domain does not appear in the list, click the Add Email Domain button to add the domain to the list. Guidewire PolicyCenter will use the selected email domain to route federation to a particular Single Sign-On (SSO) Identity Provider Server and within the Hyland CIC Platform.
15.  Click Next Step. The Settings page is displayed.
16.  Select Enabled.
17.  Click Next Step. The Review page is displayed.
18.  Review the information displayed and click Add Identity Provider. The identity provider is added to the Identity Providers list.
19.  Create a new User Group in CIC whose name matches the SSO domain. For full details on how to create a User Group, see Configuring User Groups.

## Single Sign On Configuration

You must create the following user groups in your single sign-on platform: Guidewire Doc Security - None, Guidewire Doc Security - Sensitive, Guidewire Doc Security - Unrestricted, and Guidewire Doc Security - Internal Only. These user groups will match the user groups in the Hyland Content Innovation Cloud which are created by default during environment provisioning.

Because the user groups in the Hyland Content Innovation Cloud are configured with a Group Type of external, they will be controlled by your single sign-on platform. You must then assign users to the document security user groups based on their roles and permissions. For example, if a user should be able to view Sensitive documents in Guidewire PolicyCenter, then they will need to be added to the Guidewire Doc Security - Sensitive user group in your SSO platform. See your SSO platform's documentation for detailed information on creating user groups and assigning users.

## Updating Existing Users For Single Sign On Sso

After configuring an administrator user for SSO, you must configure all other user accounts you wish to use SSO. For information on updating existing users in PolicyCenter, see your Guidewire PolicyCenter documentation.

To configure existing users for SSO:

1.  Log in to PolicyCenter as the administrator user configured for SSO. See Configuring an Administrator User for Single Sign-On (SSO) for information on configuring an administrator user for SSO.
2.  Search for the users that require updating.
3.  Update the user name for each existing user to use the format of an email address with the correct email domain enabled for SSO.
4.  Update any additional information for each existing user as required, then click on the Update button to save each user.
5.  Log in to the SSO Identify Provider as an administrator.
6.  For each configured user account, create a new account with the same user name in your SSO Identify Provider. For information on creating new users in your SSO Identify Provider, see your SSO Identity Provider's documentation.
7.  Assign each of the users modified in step 3 to the appropriate SSO group.
8.  Enable SSO for those users and save all changes.

## Updating Runtime Properties

Updating runtime properties can be performed within Guidewire PolicyCenter. See your Guidewire PolicyCenter documentation for information on updating runtime properties.

The following properties are used by Hyland for Guidewire PolicyCenter:

| Property Name | Description | Value |
| --- | --- | --- |
| ContentApi_BaseUrl | This property stores the URL that connects PolicyCenter to the instance of UCEB, which is the backend that connects Guidewire to the CIC repository. Do not modify this property unless directed to do so by a Hyland Administrator. | https://api.uceb.app-intel.experience.hyland.com |
| ContentApi_ChunkSizeBytes | This property controls the size (in bytes) of the chunks that are created when uploading a document to the CIC repository. You should only edit this property if you experience network performance issues. | The default value is 5242880. You cannot set this value lower than 5000000 or higher than 10000000. |
| ContentApi_MaxAttempts | This property controls the number of times Hyland for Guidewire PolicyCenter will attempt to send document upload information from PolicyCenter to the CIC repository before marking the upload as failed. You should only edit this property if you experience network performance issues. | The default value is 7. You cannot set this value lower than 1 or higher than 10. |
| ContentApi_OAuthScopes | This property stores the authentication scopes that are required to obtain an access token. You should only edit this property if you experience authentication issues. | environment_authorization uceb hxp.nucleus.account Note: It is recommended to remove the hxpr scope if it is listed here. This scope is intended for System Administrators and should not be used within the PolicyCenter application. |

## Viewing Document Properties

When a document is uploaded to the Hyland CIC platform, it contains specific properties which are accessible by clicking the Information button within the HX document viewer. The information stored in these properties is determined based on the values selected when the document is uploaded.

The following properties are available in the Main Properties section:

- Category
- Contributors
- Created
- Creator
- Filename
- File Size
- Last Contributor
- Last Modified
- Mime Type
- Path
- Title

The following properties are available in the Other Properties section:

- Account_Number
- Author
- Doc_Name
- Document_Security_Type
- Document_Type
- File_Type
- IS_FROM_GUIDEWIRE
- Policy_Number
- Status
- Sys ID
- User_I_D

Note: Because a service user account is used to upload Guidewire documents to the Hyland CIC platform, the following properties remain blank by default:
- Creator
- Last Contributor
- Contributors

The Guidewire user who uploaded the document is stored in the User_I_D property in the Other Properties section.
