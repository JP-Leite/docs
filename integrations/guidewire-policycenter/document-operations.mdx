## Connecting Cic Document Upload To Guidewire

You must configure your CIC Document Upload feature to connect to your Guidewire instance within the Studio Modeler application. See the Hyland Automate Modeler documentation for general information on using the Studio Modeler application.

To configure your CIC Document Upload feature to connect to your Guidewire instance:

1.  In the Studio Modeler application, click on the CIC Document Upload project. The project window is displayed.
2.  Click the Upload doc to Guidewire PC process in the Process Automation pane on the left. The project page is displayed.
3.  Click Edit Process Variables in the Properties pane on the right. The Process variables page is displayed.
4.  Locate the gwRestAPI variable and click the corresponding Edit variable button. The Edit variable page is displayed.
5.  In the Default value section, enter the url for your Guidewire instance in the Provide a value field. For example: https://your-instance.guidewire.net:443/rest/
6.  Click Save. The Process variables page is displayed.
7.  Click Close. The project page is displayed.
8.  Click Save (the cloud icon at the top bar of the pane) to save the project.
9.  Select the Release action from the bar above the project window to release a new version of the project.
10.  Open the Studio Admin application.
11.  Navigate to Devops > Application Instances. The Application Instances page is displayed.
12.  Find your CIC Document Upload project and click Upgrade to upgrade it and update the variable in your application instance. See the Hyland Automate Modeler documentation for general information on upgrading a project.

## Obtaining The Viewer Base Url Value

In most cases the Viewer_BaseUrl runtime property will need to be edited. The URL required for this property can be obtained from your Hyland for Guidewire PolicyCenter application within the Studio Admin application.

To obtain the base viewer URL, perform the following steps:

1.  Using the assigned administrator account of your CIC instance, log in to the CIC Administration Portal.
2.  Click Studio Admin. The Studio Admin application is displayed.
3.  Navigate to DevOps > Application Instances. See the Hyland Automate Administration documentation for general information on using the Studio Admin application.
4.  Find your Hyland for Guidewire PolicyCenter application and click the HX Viewer link in the Workspace column. A new page is displayed with the HX viewer URL in the address bar. This URL represents the base of the Viewer_BaseUrl runtime property. For example: https://key-abcdefg-123a-456789.studio.experience.hyland.com/guidewire-claimcente-aaaa-0000/ui/hx-viewer-abcde/#/
5.  Copy the URL displayed in the address bar.
6.  Append /default/documents/ to the base URL.  For example: https://key-abcdefg-123a-456789.studio.experience.hyland.com/guidewire-claimcente-aaaa-0000/ui/hx-viewer-abcde/#/default/documents/
7.  Set the Viewer_BaseUrl runtime property to the modifed URL. See for more information.

## Uploading A Document To Guidewire Policycenter From A Cic Repository

You can upload documents to Guidewire PolicyCenter directly from your CIC repository. See the Hyland Workspace documentation for general information on uploading a document to the CIC repository. Documents can be uploaded by starting a new process, or by uploading directly to the folder associated with the account number. Each document must be linked to a PolicyCenter item (an account or a policy). When linked to an account, the document will be visible within the account in PolicyCenter. When linked to a policy, the document will be visible within both the policy and its parent account in PolicyCenter.

To upload a document to Guidewire PolicyCenter from a CIC repository:

1.  Navigate to the Workspace application created after deploying the project.
2.  Upload a document to CIC manually, or by using a specific policy folder.
- To upload a document manually:
1. Log in to your CIC application and open Workspace.
2. Click the My Tasks > Start Process button. The New Process window is displayed.
3. Select the Upload doc to Guidewire PC process and click the Start Process button. If necessary, a new folder named unassigned will be created to temporarily store the document being uploaded.
4. Navigate to Queued Tasks and select the Upload Document/Set Metadata task. A form is displayed.
5. Click the Claim button. The form fields become editable.
6. Click the Attach button. The Select a file window is displayed.
7. Select the Local storage tab and upload a document by dragging and dropping the file into the folder or by clicking the Upload button.
8. Click Attach.
9. Enter text in all desired fields. Required fields are marked with an asterisk.
- To upload a document to a specific account folder:
1. Log in to your CIC application and open Workspace.
2. Open the hfgw-pc-documents folder.
3. Open the folder for the desired account number.
4. Upload a document by dragging and dropping the file into the folder or by clicking the File Upload > Upload File button. An Uploading file window is displayed.
5. Select the document from the Unassigned Content Type list. The Location & Category pane is displayed on the right.
6. Select hfgw-pc-policy-doc-type from the Content Type drop-down list and click Save.
7. Click Submit. The file is uploaded and a new task is created.
8. Navigate to Queued Tasks and select the Upload Document/Set Metadata task. A form is displayed.
9. Click the Claim button. The form fields become editable.
10. Enter text in all desired fields. Required fields are marked with an asterisk. The Policy Number field is automatically populated with the policy number from the selected folder.Note: The hfgw-pc-policy-doc-type content type is the only supported content type for this type of upload.
3.  Click Complete. The document is uploaded to the CIC repository and is also viewable within Guidewire PolicyCenter. For information on viewing the uploaded document, see Viewing Documents in PolicyCenter or Viewing Documents in CIC.Note: If the account number or policy number you specified does not exist within the PolicyCenter instance, the document is not uploaded and a new task is generated to update the account or policy number.

## Viewing Documents In Cic

You can view the PolicyCenter documents stored in your ECM repository directly from the CIC platform. See the Hyland Workspace documentation for detailed information on retrieving and viewing documents within the CIC platform.

To view PolicyCenter documents in CIC:

1.  Launch the Guidewire Workspace within the Hyland CIC Platform.
2.  If prompted to log in, enter the appropriate credentials. If your solution is configured to use single sign-on (SSO), you can log in using your PolicyCenter credentials. If your solution is not configured to use single sign-on (SSO), you can log in using your Hyland CIC Platform credentials.
3.  Open the folder with the name that matches the account number for the document you want to view.  Note: All PolicyCenter documents uploaded to CIC (including policy documents) are stored in the folder for the document's account number. The document's policy number is stored as a property.
4.  Within the folder, click the document you wish to view. The document is displayed in a separate window in the HX document viewer. For information on working with a document within your CIC repository, see the CIC documentation.
5.  Optional: If you want to show or hide the document's properties, click the click the information button in the upper right corner of the viewer. For more information on document properties, see Viewing Document Properties.
6.  Optional: If you want to download the document, click the download button in the upper right corner of the viewer.

## Viewing Documents In Policycenter

Depending on how your system is configured, you may be able to view documents using your ECM repository's document viewer from the PolicyCenter Document Results List.

Note: In order to view PolicyCenter documents, ensure that your browser is configured to allow pop-ups launched from PolicyCenter.
To view a document stored in your ECM repository using the PolicyCenter Document Results List:

1.  Open a policy or acount within the PolicyCenter application.
2.  Click the Documents tab.  A list of related documents in your ECM repository is displayed in PolicyCenter.
3.  Click the name of the document you want to view. The document is displayed in a separate window in your ECM repository's document viewer. For information on working with a document within your ECM repository, see your ECM repository's documentation.Note: Depending on your system's configuration, you may be prompted to log in using your ECM repository credentials (for example, your CIC credentials). If single sign-on is configured for your solution, you may only need to enter the email address for your CIC user to log in.
4.  Optional: If you want to show or hide the document's properties, click the click the information button in the upper right corner of the viewer. For more information on document properties, see Viewing Document Properties.
5.  Optional: If you want to download the document, click the download button in the upper right corner of the viewer.
