---
title: "Hosting Onbase Connector On A Custom Website"
---

If your company policy prohibits using port 80, you must create a new website in Internet Information Services (IIS) Manager and provide a port of your choice.

The new website must be created before running DeployOBConnector.ps1 described in Starting the OnBase Connector. To create a custom website in Internet Information Services (IIS) Manager:

1.  Add a website in Internet Information Services (IIS) Manager.  Note: To add a new website, see the IIS Manager section on the Microsoft documentation site [https://learn.microsoft.com/](https://learn.microsoft.com/).
Provide the following details:

-
Site Name: The custom website name.

-
Physical Path: The installation directory path.

-
Port: A custom port number, such as 82. By default, IIS Manager uses port 80.

2.  Open the applicationConfig.json file in a plain text editor, such as NotePad. The file is located in the Configuration folder in the root installation directory, such as C:\OnbaseConnectorSetUp\Configuration\applicationConfig.json
3.  In the  OnBase Connector section, update WebHostBaseUrl, GatewayBaseUrl, NucleusIntegrationBaseUrl, and DocumentImportBaseUrl with the port number of your custom IIS website to use the selected port.
For example:

```
"WebHostBaseUrl": "http://localhost:82/OnbaseConnectorWebHost/",
"GatewayBaseUrl": "http://localhost:82/OnbaseConnectorGateway/",
"NucleusIntegrationBaseUrl": "http://localhost:82/OnbaseConnectorNucleusIntegration/",
"DocumentImportBaseUrl": "http://localhost:82/OnbaseConnectorDocumentImport/",
```

4.  Save the applicationConfig.json file.
