---
title: "Encrypting the Secret Keys"
description: "Secure your OnBase Connector credentials by encrypting them using the built-in CLI tool."
---

Encrypting secret keys protects sensitive credentials (database connection string, Client ID, Client Secret, and UnityAPI credentials) from unauthorised access. **This step is required before starting the connector.**

---

## Encryption Steps

<Steps>
  <Step title="Open a Terminal as Administrator">
    Run **Command Prompt** or **Windows PowerShell** with administrator privileges.
  </Step>
  <Step title="Navigate to the CLI Folder">
    Use `cd` to navigate to the `ContIntOnBaseConnector.CLI` folder in your installation directory:

    ```bash
    cd C:\OnbaseConnectorSetUp\OnbaseConnector\ContIntOnbaseConnector.CLI
    ```

    Press **Enter**.
  </Step>
  <Step title="Run the Encrypt-Secrets Command">
    Run the encryption command:

    <Tabs>
      <Tab title="Command Prompt">
        ```
        ContIntOnbaseConnector.CLI encrypt-secrets
        ```
      </Tab>
      <Tab title="PowerShell">
        ```
        .\ContIntOnbaseConnector.CLI encrypt-secrets
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Enter the Database Connection String">
    When prompted, enter your OnBase Connection String and press **Enter**.

    Example:
    ```
    Data Source=.\SQLEXPRESS; database=OnBase; User Id=hsi; Password=wstinol; Encrypt=Optional;
    ```

    The connection string can be found in the **Web Application Management Console** under **Connection Strings → View Contexts**.

    When asked **"Do you want to test the connection?"**, type `y` and press **Enter**.
  </Step>
  <Step title="Enter the Client ID">
    When prompted, enter your **Client (Application) ID** and press **Enter**.

    Example: `sc-cdf19c81-0962-424d-bff4-678a20e75aa9`
  </Step>
  <Step title="Enter the Client Secret">
    When prompted, enter your **(Client) Secret** and press **Enter**.

    When asked **"Do you want to test the credentials?"**, type `y` and press **Enter**.

    A `Got token from IDP` message confirms that your Client ID and Client Secret are correct.
  </Step>
  <Step title="Enter UnityAPI Credentials">
    When prompted, enter your **UnityAPI username** and press **Enter**, then enter your **UnityAPI password** and press **Enter**.

    When asked **"Do you want to test the credentials?"**, type `y` and press **Enter**.
  </Step>
  <Step title="Verify Encryption">
    The secret keys are now encrypted. A `Logger` folder is created inside the `ContIntOnbaseConnector.CLI` folder. Check the log file for encryption status confirmation.
  </Step>
  <Step title="Close the Terminal">
    Close the terminal window.
  </Step>
</Steps>

---

## What Happens Next?

After encryption, an `AppSecrets` folder is created in the OnBase Connector installation directory containing the encrypted credentials. The connector reads from this folder at runtime — no plain-text passwords are stored in configuration files.
