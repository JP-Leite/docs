## Configuration Overview

Configuring the OnBase Connector for Content Intelligence involves three steps:

<CardGroup cols={3}>
  <Card title="Required: applicationConfig.json" icon="gear" href="/integrations/onbase/configuring-the-applicationconfigjson-file">
    Connect to your OnBase Application Server and link to your Content Innovation Cloud environment. **This step is mandatory.**
  </Card>
  <Card title="Required: Encrypt Secret Keys" icon="lock" href="/integrations/onbase/encrypting-the-secret-keys">
    Securely encrypt your database connection string, Client ID, Client Secret, and UnityAPI credentials using the CLI. **This step is mandatory.**
  </Card>
  <Card title="Optional: filterSettings.json" icon="filter" href="/integrations/onbase/configuring-the-filtersettingsjson-file">
    Restrict ingestion to specific Document Types and Keyword Values. If omitted, all documents are ingested. **This step is optional.**
  </Card>
</CardGroup>

## Pre-Configuration Checklist

Before configuring, gather the following information:

| Information | Description | Example |
| --- | --- | --- |
| **Client (Application) ID** | IDP token credential. Requires a service user and external application with the correct scopes (`hxp.integrations`, `iam.user-data.account`, `system-integrations-config`, etc.). | `sc-cdf19c81-0962-424d-bff4-678a20e75aa9` |
| **(Client) Secret** | Client credential for IDP token. Copy when creating the service user — **it is shown only once.** | `tPln1sEnhG7...VBae5G=` |
| **Environment Key** | CIC Environment Key. Found in the Knowledge Discovery URL. | `hxai-e4ea560-a906-3246-90bc-eb882443d36c` |
| **Source ID** | UUID identifying the connected repository. Created when connecting a repository in Knowledge Discovery. | `879cce2c-7ja9-9641-b256-3a95bc864d7e` |
| **SystemIntegrationId** | System integration ID between the connector and OnBase. Created when registering a system integration in the CIC Administration Portal. | `ab705aaa-a0a3-46e3-ba44-328d0321f9fc` |

<Warning>
  Do not store the Client Secret in plain text in configuration files. The encryption step will secure it for you.
</Warning>

## Configuration Steps

<Steps>
  <Step title="Configure applicationConfig.json">
    Update the `applicationConfig.json` file with your OnBase Application Server URL and CIC Ingestion Connection settings.

    [Configure applicationConfig.json →](/integrations/onbase/configuring-the-applicationconfigjson-file)
  </Step>
  <Step title="Configure defaultConfig.json">
    Update the `defaultConfig.json` file with your `SystemIntegrationId`.

    [Configure defaultConfig.json →](/integrations/onbase/configuring-the-defaultconfigjson-file)
  </Step>
  <Step title="Encrypt the Secret Keys">
    Run the CLI to encrypt your database connection string, Client ID, Client Secret, and UnityAPI credentials.

    [Encrypt Secret Keys →](/integrations/onbase/encrypting-the-secret-keys)
  </Step>
  <Step title="(Optional) Configure Filters">
    If you want to restrict ingestion to specific Document Types, configure `filterSettings.json`.

    [Configure Filters →](/integrations/onbase/configuring-the-filtersettingsjson-file)
  </Step>
</Steps>
