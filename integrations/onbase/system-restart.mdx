---
title: "System Restart"
description: "How to restart the OnBase Connector for Content Intelligence after a system restart, shutdown, or IIS reset."
---

After a **system restart**, **server shutdown**, or **IIS reset**, the OnBase Connector for Content Intelligence must be manually restarted using the `start-subscriptions` CLI command.

---

## Steps

<Steps>
  <Step title="Open a Terminal as Administrator">
    Run **Command Prompt** or **Windows PowerShell** with administrator privileges.
  </Step>
  <Step title="Navigate to the CLI Folder">
    ```bash
    cd C:\OnbaseConnectorSetUp\OnbaseConnector\ContIntOnbaseConnector.CLI
    ```

    Press **Enter**.
  </Step>
  <Step title="Run the Start-Subscriptions Command">
    <Tabs>
      <Tab title="PowerShell">
        ```
        .\ContIntOnbaseConnector.CLI start-subscriptions
        ```
      </Tab>
      <Tab title="Command Prompt">
        ```
        ContIntOnbaseConnector.CLI start-subscriptions
        ```
      </Tab>
    </Tabs>

    On the first run, a `ContIntOnBaseConnector` folder is created inside `ContIntOnBaseConnector.Gateway`, containing a `subscriptions.json` file. On subsequent runs, `subscriptions.json` is updated.
  </Step>
  <Step title="Verify the Subscription Status">
    Open `subscriptions.json` and confirm the subscription status is `0`:

    ```json
    "status": 0
    ```

    A status of `0` indicates the subscription is active.
  </Step>
</Steps>
