---
title: "Advanced Configuration"
description: "Configure optional logging settings in defaultConfig.json to control log verbosity, file paths, and Gateway performance."
---

The `defaultConfig.json` file supports optional advanced settings for **logging** and **Gateway performance**. These settings help you capture connector behaviour and tune throughput.

The file is located at:
```
C:\OnbaseConnectorSetUp\Configuration\defaultConfig.json
```

---

## Applicable Services

Logging settings in `defaultConfig.json` apply to the following connector services:

- `ConIntOnbaseConnector.WebHost`
- `ConIntOnbaseConnector.Gateway`
- `ConIntOnbaseConnector.CLI`
- `ContIntOnbaseConnector.DocumentImport`
- `ContIntOnbaseConnector.NucleusIntegration`

---

## Logging Settings

| Key | Accepted Values | Description |
| --- | --- | --- |
| `path` | File path string | Location and filename where log files are saved. Example: `"path": "Logger\\log.log"` |
| `MinimumLevel` | `Verbose`, `Debug`, `Information`, `Warning`, `Error`, `Fatal` | Minimum severity level for log events to be recorded. Default: `"MinimumLevel": { "Default": "Verbose" }`. See [MinimumLevel Logs](/integrations/onbase/minimumlevel-logs) for details. |
| `rollingInterval` | `Hour`, `Day` | How frequently log files are automatically rotated. Example: `"rollingInterval": "Hour"` |

---

## Gateway Performance Settings

These settings apply to the **Gateway** section in `defaultConfig.json` and control throughput:

| Key | Accepted Values | Description |
| --- | --- | --- |
| `BatchSize` | Any number up to `100` | Number of events sent to a subscriber at a time. Do not exceed 100. Example: `"BatchSize": 100` |
| `BlockSize` | Any number | Number of records fetched from the database per processing cycle. Example: `"BlockSize": 500` |
| `Delay` | Any number (seconds) | Time (in seconds) the logger waits before reading data from the database. Example: `"Delay": 2` |
