---
title: "Minimumlevel Logs"
---

The MinimumLevel setting filters out log messages that are below the specified severity threshold, ensuring that only messages at that level or higher are recorded or displayed.

The default value for MinimumLevel is Verbose. With this setup, information level events and higher are processed.

The following table describes the available logging levels:

| Log Level Value | Description |
| --- | --- |
| Verbose | Log messages that provide extensive detail. They are useful for debugging but rarely enabled in production environments due to their volume. |
| Debug | Log messages that are used for internal system events. They provide insight into how processes unfold. |
| Information | Log messages that track system events related to its core responsibilities and functions. |
| Warning | Log messages that indicate that a service is degraded, endangered, or behaving outside expected parameters. |
| Error | Log messages that are used when functionality is unavailable or expectations are broken. |
| Fatal | Log messages that describe an unrecoverable application, system crash, or catastrophic failure that requires immediate attention. |
