## Deploying A Project

This section has information on deploying the hfs-configurations project you released earlier to the subscribed Content Innovation Cloud (CIC) environments. To deploy a new release, you must use the Hyland Automate Administration application.

Released projects can be deployed from the Project Releases menu in Hyland Automate Administration. Click the three dots adjacent to the hfs-configurations project and select Deploy.

When deploying a released project, the user is presented with various configuration settings that may vary depending on the contents of the project.

Tip: Projects are referred to as applications when they are deployed.
For more information on these various configuration settings, see the Deploying Projects chapter in the Hyland Automate Administration documentation.

Select the Previous and Next buttons to navigate between the configuration pages. The current step is highlighted, and the completed steps are marked with a tick.

After completing the required configuration settings, click Deploy to close the configuration wizard and deploy the application.

## Installing The Hyland For Salesforce Package

This section contains information on how to install the Hyland for Salesforce package.

To install the Hyland for Salesforce package:

1.  Open a web browser and copy paste the following URL in the address bar:
https://login.salesforce.com/packagingSetupUI/ipLanding.app?apvId=04tV1000000R4SXIA0

Note: If you are installing to a sandbox organization, replace the above URL with the following: https://test.salesforce.com/packagingSetupUI/ipLanding.app?apvId=04tV1000000R4SXIA0
2.  Log on to Salesforce with your credentials. The Install Hyland for Salesforce page is displayed.
3.  From the Install Hyland for Salesforce page, select from the available options:

| Option | Description |
| --- | --- |
| Install for Admins Only | Allows you to install the Hyland for Salesforce package for system administrators only. |
| Install for All Users | Allows you to install the Hyland for Salesforce package for all users. |
| Install for Specific Profiles | Allows you to install the Hyland for Salesforce package for specific profiles. |

4.  Click Install.
5.  After installation is complete, click Done.

## Introduction

Hyland for Salesforce provides a web-based solution to manage critical, unstructured content by utilizing Hyland's modern Content Innovation Cloud and Enterprise Content Management (ECM) capabilities with Salesforce's Customer Relationship Management (CRM) capabilities.

Hyland for Salesforce enables you to access and manage your documents directly from Salesforce. You can perform key tasks such as capturing, viewing, and configuring Hyland Content or your documents with contextual information (or Keyword Values) to categorize them meaningfully and retrieve them. These tasks can be easily accomplished with a few clicks in a pane within Salesforce so that they become an extension of the Salesforce object, providing greater efficiency in managing your records.

## Key Features

Hyland for Salesforce offers the following features:

-
Accelerate user productivity and adoption by eliminating switching between Hyland's Content Innovation Cloud and Salesforce.

-
Enable seamless importing of documents through an intuitive drag-and-drop interface for supported files.

-
Access vital documents instantly from any location.

-
Enhance retrieval options by configuring and associating properties with supported files.

-
Reduce processing delays and loss of information, as the risk of duplicating critical documents to associate them with Salesforce objects is mitigated.

-
Expedite and improve business decision-making by delivering immediate access to relevant documents.

-
Improve data accuracy and reduce re-keying of information by pulling data directly from Hyland's Content Innovation Cloud.

-
Link one document to multiple Salesforce objects.

## Registering Hyland For Salesforce As An External Application

This section has information on how to register Hyland for Salesforce as an external web server application to enable integration with the Content Innovation Cloud, and obtain the Application ID and Secret. The Application ID and Secret are required to configure the External Auth Identity Provider in Salesforce.

Note: For more information on creating an External Application using the Content Innovation Cloud Administration Portal, see the External Systems chapter in the Content Innovation Cloud Administration Portal documentation.
To register Hyland for Salesforce as an external application:

1.  Open the Content Innovation Cloud Administration Portal.
2.  Select the External Systems tab under the banner. The External Applications page is displayed.
3.  Click Create External Application. The Create External Application wizard is displayed.
4.  Select Web Server Application.  Note: Remember to register Hyland for Salesforce as a web server application as it calls Content Innovation Cloud APIs in the context of an interactive (human) user.
5.  Click Next Step. The Application Name page is displayed.
6.  Enter a name for the application in the Application Name field, and then click Next Step. The Redirect URIs page is displayed.
7.  On the Redirect URIs page, enter the following URIs as applicable for your environment:
- Developer or Enterprise org: https://login.salesforce.com/services/extidp/callback
- Sandbox org: https://test.salesforce.com/services/extidp/callback
After a user signs in successfully to Hyland for Salesforce, they are redirected to one of these URIs.

8.  Click Next Step. The Post Logout Redirect URIs page is displayed.
9.  On the Post Logout Redirect URIs page, enter the following URIs, as applicable for your environment:
- Developer or Enterprise org: https://login.salesforce.com/services/extidp/callback
- Sandbox org: https://test.salesforce.com/services/extidp/callback
After a user logs out successfully from Hyland for Salesforce, they are redirected to one of these URIs.

10.  Click Next Step. The Allowed Scopes page is displayed
11.  From the Allowed Scopes drop-down list, select the following scopes:
iam.scim openid profile profile.attributes hfs hxp environment_authorization offline_access uceb hxp.nucleus.account

Hyland for Salesforce is allowed to request a grant for the scopes you specify.

Note: For more information on each of these scopes, see the Scopes for External Applications chapter in the Content Innovation Cloud Administration Portal documentation.
12.  From the Environment drop-down list, select the environment Hyland for Salesforce should be restricted to. The Application drop-down list is displayed.
13.   From the Application drop-down list, select the subscribed application to link to Hyland for Salesforce.
14.  Click Next Step. The Review page is displayed.
15.  Review the settings for the external application.
To return to a previous step, either select the step from the left side of the wizard, or click Previous Step.

16.  Click Create External Application to confirm the settings and register the external application. The Summary page displays the auto-generated Application ID and Client Secret for the external application.
17.  Copy the Application ID and Secret values to a secure location.
Hyland for Salesforce requires these values in order to obtain access tokens from the Content Innovation Cloud.

Note: The secret value will not be shown again after the Create External Application wizard is closed. If necessary, you can reset the secret as described under Resetting the Secret for an External Application chapter in the Content Innovation Cloud Administration Portal documentation.
18.  Click Close to return to the External Applications page.

## Uninstalling The Hyland For Salesforce Package

This section contains information on how to uninstall the Hyland for Salesforce package.

To uninstall the Hyland for Salesforce package:

1.  Open a web browser and log on to the Salesforce environment with your credentials. The Home page is displayed.
2.  From the top-right, click Setup. The Home page is displayed in a new browser tab.
3.  In the Quick Find box, type Installed Packages and then click Installed Packages from the search results displayed on the left. The Installed Packages page is displayed.
4.  Search for Hyland For Salesforce.
5.  Under the Custom Object Data section, select the following options:
-
Do not save a copy of this package's data after uninstall

-
Yes, I want to uninstall this package and permanently delete all associated components.

6.  Click Uninstall.

## Upgrading A Project

This section, intended for system administrators, explains how to upgrade the hfs-configurations project and include new Content Types you ce. Use Hyland Automate Administration to prepare the new release.
The upgrade process displays the same configuration as deploying a project with an additional section of Runtime Version, which shows the Runtime Version of the updated application. The only field that cannot be changed when upgrading is the application name.

Note: For more information on upgrading an application, see the Upgrading chapter in the Hyland Automate Administration documentation.
To upgrade a project:

1.  Log in to the Hyland Automate Administration App.
2.  Expand Devops from the left pane and select Application Instances.
3.  Click the three dots next to hfs-configurations.
4.  Select Upgrade.
5.  Select the details you want to upgrade and click Upgrade.  Note: Regularly updating your project to the latest runtime version is a best practice. For specific instructions on how to update the runtime versions of applications, see the Updating the Runtime Version of Applications chapter in the Hyland Automate Administration documentation.
