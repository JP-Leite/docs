## Adding Components

This sections explains how to add the components Document List View or hfsRecordDLPage and Document Capture or hfsCaptureApp.

These components control the visibility of the Document List window and the Upload and Import window within Hyland for Salesforce. If these components are added to the page layout, the Document List window and the Upload and Import window are visible.

See the following for specific instructions on these topics:

-
Adding the Document List View Component

-
Adding the Document Capture Component

## Adding The Document Capture Component

This sections explains how to add the Document Capture hfsCaptureApp component.

You must add the Document Capture or hfsCaptureApp component to the record detail page of of the corresponding Salesforce object. This ensures users can view related documents in the context of the specific Salesforce object record.
To add the Document Capture component:

1.  Log on to Salesforce with your credentials. The Home page is displayed.
2.  From the top-right, click Setup. The Home page is displayed in a new browser tab.
3.  In the left pane, select Platform Tools | Objects and Fields | Object Manager.  The Object Manager page is displayed.
4.  On the Object Manager page, navigate to the Salesforce object for which you created a mapping.
5.  To edit the lightning record page for this Salesforce object, click Edit. The Lightning App Builder page is displayed.
6.  From the left pane, scroll to Custom Components, select the hfsCaptureApp component and then drag it to the desired area on the Salesforce record detail page.
7.  In the Height(in pixels) box, enter 750.
8.  Click Save.

## Assigning Permission Sets

This section explains how to assign the permission sets to users according to their role or function. These permissions sets include the Hyland For Salesforce External Credential Principal Access, Hyland For Salesforce Users, and Hyland For Salesforce Admin Users.

Note: The Hyland For Salesforce Users and Hyland For Salesforce Admin Users permission sets do not need to be manually created, as they are predefined (hardcoded) within the Hyland for Salesforce.
To enable access and configuration capabilities in Hyland for Salesforce, ensure the following permissions sets are assigned to the correct roles:

| Permission Set | Role |
| --- | --- |
| Hyland For Salesforce External Credential Principal Access | Users who need to view and capture documents using Hyland for Salesforce. |
| Salesforce Administrators or system administrators who need to access and configure the Application Configurations and Solution Configurations sections within Hyland for Salesforce. |
| Hyland For Salesforce Users | Users who need to view and capture documents using Hyland for Salesforce. |
| Hyland For Salesforce Admin Users | Salesforce Administrators or system administrators who need to access and configure the Application Configurations and Solution Configurations sections within Hyland for Salesforce. |

See the following sections for more information on these topics:

-
Assigning the Hyland For Salesforce External Credential Principal Access Permission Set

-
Assigning the Hyland For Salesforce Users Permission Set

-
Assigning the Hyland For Salesforce Admin Users Permission Set

## Assigning The Hyland For Salesforce Admin Users Permission Set

This section explains how to assign the Hyland For Salesforce Admin Users permission set to users according to their role or function.

To ensure proper access and functionality in Hyland for Salesforce, assign the Hyland For Salesforce Admin Users permission set to Salesforce administrators or system administrators who need to access and configure the Application Configurations and Solution Configurations sections within Hyland for Salesforce. This permission set does not need to be manually created, as it is predefined (hardcoded) within Hyland for Salesforce.
To assign the Hyland For Salesforce Admin Users permission set:

1.  Log on to Salesforce with your credentials. The Home page is displayed.
2.  From the top-right, click Setup. The Home page is displayed in a new browser tab.
3.  In the Quick Find box, type Permission Sets and then click Permission Sets from the search results displayed on the left. The Permission Sets page is displayed.
4.  Locate Hyland For Salesforce Admin Users and click it. The Hyland For Salesforce Admin Users page is displayed.
5.  Click Manage Assignments. The Current Assignments page is displayed.
6.  Click Add Assignment. The Select Users to Assign page is displayed.
7.  Select one or more check boxes to the left of each user's name and click Next. The Selected Users page is displayed.
8.  On the Selected Users page, click Assign. If the permission is assigned successfully, a confirmation message is displayed. Else, a failure message is displayed.
9.  Click Done. The Hyland For Salesforce Admin Users Current Assignments page is displayed.

## Assigning The Salesforce Content User User Group To The Salesforce Content User

To assign the Salesforce Content User User Group to the Salesforce Content User Role:

1.  Open the Administration Portal.
2.  Select the Account tab under the banner. The Account Details page is displayed.
3.  From the left pane, click Applications.  Note: If the Hx account includes multiple environments, ensure the correct environment is selected from the left pane.  A list of applications is displayed on the right pane.
4.  Click Hyland for Salesforce and then click User Rights.
5.  Click Manage next to Salesforce Content User Role.
6.  Do one of the following:
- In the search box, enter Salesforce Content User and select it.
- From the list of User Groups under Unassigned, search for Salesforce Content User and select it.
7.  Click + next to Salesforce Content User. The Salesforce Content User User Group is displayed under Assigned.
8.  Click Save.

## Configuring Authentication Settings In Salesforce

This section has information on how to configure an External Auth Identity Provider in Salesforce by using the Application ID and Client Secret obtained while registering Hyland for Salesforce as an external web server application. This configuration enables Hyland for Salesforce to integrate with the Hyland Content Innovation Cloud.

Note: Before you proceed, you must have the Application ID and Client Secret and complete the instructions described under Registering Hyland for Salesforce as an External Application.
To configure an External Auth Identity Provider in Salesforce:

1.  Log on to Salesforce with your credentials. The Home page is displayed.
2.  From the top-right, click Setup. The Home page is displayed in a new browser tab.
3.  In the Quick Find box, type Named Credentials and then click Named Credentials from the search results displayed on the left. The Named Credentials page is displayed.
4.  Click the External Auth Identity Providers tab and then click HylandIAM. The HylandIAM page is displayed.
5.  From the top-right, click Edit. The Edit HylandIAM popup is displayed.
6.  In the Client ID field, enter the value for Application ID you obtained while registering Hyland for Salesforce as an External Application.  Note: For instructions on obtaining the Application ID, see Registering Hyland for Salesforce as an External Application.
7.  In the Client Secret field, enter the value for Client Secret you obtained while registering Hyland for Salesforce as an External Application.  Note: For instructions on obtaining the Client Secret, see Registering Hyland for Salesforce as an External Application.
8.  Click Save. A confirmation message is displayed if the changes are successfully saved and the HylandIAM page is displayed.

## Creating Content Types

This section, intended for system administrators, contains information on creating new Content Types that define the content you can use with Hyland for Salesforce. The Content Types you create define the properties for the different file types that are supported by Hyland for Salesforce.

Use Hyland Automate Modeler to create new Content Types. You can use built-in Content Types to use default metadata and properties, or create custom Content Types using Schemas and Mixins to define properties as required.
For more information on Content Types, see the Content Types chapter in the Hyland Automate Modeler documentation.

To create the Content Type required for Hyland for Salesforce:

1.  Sign in to the Hyland Automate Modeler application and open the hfs-configurations project.
2.  Do one of the following to create the content type:
- Click Create > Content Type.
- Click the three dots next to the Content Types drop-down menu in the left-hand pane, then select Create.
You can also upload a JSON file with the content type. For more instructions, see the Uploading Project Elements chapter in the Hyland Automate Modeler documentation.

The Create Content Type window is displayed.

3.  Enter the newly created content type name in the Content Type name field.
4.  Add an optional description in the Content Type description field.
5.  From the drop-down Parent Content Type field, select the parent content type. The parent content type properties will be inherited by the new content type. The default selection is None, which means that no content type properties will be inherited by the newly created content type.
The default content types that come built-in in the Hyland Automate Modeler application are under the Default Content Types section. They are as follows:

| System Content Type | Description |
| --- | --- |
| SysFile | Creates a content type that behaves like a file within your application. |
| SysFolder | Creates a content type that behaves like a folder within your application. |

Any custom-made content type can be selected under the Custom Content Types section.

Note:
If you select None as the Parent Content Type, you must add a Schema or a Mixin to it later. For more information on Schemas and Mixins, see the Schemas and Mixins chapters in the Hyland Automate Modeler documentation.

6.  Click Create Content Type. The created content type is opened in the application as a new tab.
7.  In the Schemas section, use the + icon to add schemas to your content type. If you used a parent content type that already had schemas assigned, they are listed as Inherited schemas in this section.
If you want to remove an already added schema, use the - icon on the right.

Note:
You cannot remove the inherited schemas.

For more information on schemas see the Creating and Configuring Schemas chapter in the Hyland Automate Modeler documentation.
8.  In the Mixins section, use the + icon to add mixins to your content type. If you used a parent content type that already had mixins assigned, they are listed as Inherited mixins in this section.  Note:
-
Ensure the hfs-lob-mixin Mixin is added while selecting the Content Type. This enables the new Content Type to inherit the properties of this Mixin. These properties are required for Hyland for Salesforce.

-
You cannot remove the inherited mixins.

If you want to remove an already added mixin, use the - icon on the right.

The default mixins available in the system are:

| Mixin | Description |
| --- | --- |
| SysFilish | Allows the content with this mixin to behave like a file and have its properties. |
| SysFolderish | Allows the content with this mixin to behave like a folder and have its properties, including having child documents. |
| SysFullTextIndexable | Allows the text within the content with this mixin to be fully indexable. |
| SysVersionable | Allows the content with this mixin to be versioned. |

For more information on mixins, see the Creating Mixins chapter in the Hyland Automate Modeler documentation.
9.  Once editing is done, use the Save Content Type cloud icon at the top bar to save the created content type. A blue window reading Content Type saved is displayed at the bottom. The new content type is created and saved.

## Creating The Hyland For Salesforce External Credentials Principal Access Permiss

This section explains how to create the necessary Salesforce permission sets to enable integration with Hyland's content repository.

While connecting Hyland's content repository with Salesforce, the Named Credentials and External Credential Principals are used to authenticate connections from Salesforce. To access Named Credentials and External Credential Principals, you must create the Hyland For Salesforce External Credentials Principal Access permission set.
To create the Hyland For Salesforce External Credentials Principal Access permission set:

1.  Log on to Salesforce with your credentials. The Home page is displayed.
2.  From the top-right, click Setup. The Home page is displayed in a new browser tab.
3.  In the Quick Find box, type Permission Sets and then click Permission Sets from the search results displayed on the left. The Permission Sets page is displayed.
4.  From the top-left on the Permission Sets page, click New. The Enter permission set information section is displayed.
5.  In the Label field, enter Hyland For Salesforce External Credentials Principal Access and click Save.  Note: The API Name field is automatically populated.  The details are saved and the Permission Sets Overview page is displayed.
6.  On the Permission Sets Overview page, click Object Settings. The Object Settings page is displayed.
7.  Locate User External Credentials and click on it. The User External Credentials page is displayed.
8.  Click Edit. The Object Permissions page is displayed.
9.  Select the check box next to Modify All Recordsand then click Save. The User External Credentials page showing enabled permissions is displayed.
10.  Navigate to the Permissions Set Overview page and click External Credential Principal Access. The External Credential Principal Access page is displayed.
11.  Click Edit. The External Credential Principal Access page is displayed.
12.  Under the Available External Credential Principals column, select HylandIAMCredential - Default and then click Add.  HylandIAMCredential - Default is displayed under the Enabled External Credential Principals column.
13.  Click Save. The External Credential Principal Access page is displayed.
