---
title: "Upgrading A Project"
---

This section, intened for system administrators, explains how to upgrade the project you created and include the new Content Types you created for a new release. To upgrade a project and then prepare a new release, use Hyland Automate Administration.Note:
-
If you have not created a project, please complete the instructions under Creating a Project.

-
For more information on upgrading an application, see the Upgrading chapter in the Hyland Automate Administration documentation.

To upgrade a project:

1.  Log in to the Hyland Automate Administration App.
2.  From the left pane, expand Devops and select Application Instances.
3.  Search for the project you created earlier and click the three vertical dots next to it.  Note: If you have not created a project, please complete the instructions under Creating a Project.  A popup menu is displayed.
4.  Select Upgrade.
5.  Select the details you want to upgrade and then click Upgrade.  Note: Regularly updating your project to the latest runtime version is a best practice. For specific instructions on how to update the runtime versions of applications, see the Updating the Runtime Version of Applications chapter in the Hyland Automate Administration documentation.
