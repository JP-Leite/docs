Use the Add Identity Provider wizard to add an external identity provider to your Content Innovation Cloud account.

If your account requires custom attributes for user profiles, please contact your Hyland support representative. Custom user attributes must be added to your account before they can be mapped for an external identity provider.Note: When configuring an Identity Provider, you must enter specific preconfigured values. These values are generated when you create a new Entra ID application for SAP IAS. If you don’t have these values, use placeholder (dummy) values. Remember to replace the dummy values with the correct ones later. For more information on creating a new Entra ID application for SAP IAS, see Creating an Entra ID application for SAP IAS.
To add an external identity provider:

1.  Open the SuccessFactors.
2.  Select the Identity tab below the application header. The Users page is displayed.
3.  In the sidebar, click Identity Providers. The Identity Providers page is displayed.
4.  Click Add Identity Provider in the upper-right corner of the page. The Add Identity Provider wizard is displayed.
5.  On the Information page, enter the following information:

| Setting | Description |
| --- | --- |
| Identity Provider Name | The friendly name for this identity provider. The Content Innovation Cloud sign-in page displays this value as an authentication option for users whose email domains are assigned to this identity provider. |
| Entity ID | The globally unique SAML identifier of the identity provider. To obtain this, use the App Federation Metadata Url details available while creating a new Entra ID application. For more information, see Creating an Entra ID application for SAP IAS and Quickstart: Create and assign a user account chapter in the Application management documentation for Microsoft Entra ID |
| Metadata URL | The URL from which SAML metadata will be collected.  SAML metadata must be provided through an internet-accessible URL. You cannot import the metadata as a file. To obtain this, use the App Federation Metadata Url details available while creating a new Entra ID application. For more information, see Creating an Entra ID application for SAP IAS and Quickstart: Create and assign a user account chapter in the Application management documentation for Microsoft Entra ID |
| Minimum Assertion Signature Algorithm | The minimum signature algorithm allowed to process an assertion. Select RSA-SHA256. |

6.  Click Next Step.
7.  On the Bindings page, specify whether this identity provider should override the default binding for authentication requests or assertions.
| Setting | Description |
| --- | --- |
| Override authentication request binding | Select to override the default binding for authentication requests. The default request binding is HTTP Redirect. |
| Binding Type | Select Redirect. |
| Override assertion binding | Select to request a specific binding for SAML assertions. |
| Binding Type | Select Post. |

8.  Click Next Step.
9.  On the Claim Type Mapping page, enter the SAML attribute for each claim type.
Only claim types marked with an asterisk are required.

10. **Optional: ** Map custom user attributes as needed.  Note: Custom user attributes must be added to your Content Innovation Cloud account before they can be mapped. Contact your Hyland support representative if your account requires custom user attributes.
1.  On the Claim Type Mapping page, select the Custom Attributes tab.
2.  Click Map Custom Attribute. If this option is not available, then there are no custom user attributes available to map.
3.  Select a custom user attribute from the Attribute drop-down list.
4.  In the Source Claim Name field, enter the name of the attribute as specified in SAML assertions from this identity provider. This value is case sensitive.
5.  Map additional custom user attributes as needed.
11.  Click Next Step.
12.  On the Email Domains page, specify the email domains allowed to authenticate using this identity provider.
-
Select one or more existing email domains to assign to this identity provider.

-
Click Add Email Domain to add a new email domain.

After adding a domain, you can select it for this identity provider. For information about adding email domains, see the Adding an Email Domain chapter in the Content Innovation Cloud Administration Portal documentation.

Users can authenticate using this identity provider only if their email domains match one of those selected.
13.  Click Next Step.
14.  On the Settings page, indicate whether this identity provider configuration should be enabled.
-
Choose Enabled to make the identity provider an available sign-in option as soon as the configuration is saved.

-
Choose Disabled if the identity provider should not yet be available to authenticate users for the Content Innovation Cloud.

15.  Click Next Step.
16.  Review the configuration settings you specified.
17.  Click Add Identity Provider to complete the process. The new identity provider is added to the Identity Providers list, and a confirmation message is displayed.
