---
title: "Configuring The Guid"
---

You must configure the Guidewire ClaimCenter Workspace project so that it contains the GUIDs for the Hyland Content Innovation Cloud’s security user groups.

You must have:
-
Administrator access to Guidewire ClaimCenter.

-
Administrator access to the Hyland Content Innovation Cloud.

To update ClaimCenter with the required GUIDs:

1.  Log in to the CIC Administration Portal.
2.  Navigate to Identity > User Groups. The User Groups pane is displayed.
3.  Click the Guidewire Doc Security - None user group.
4.  Copy the GUID for the user group, which is displayed at the end of the URL in the address bar. For example, if the URL in the address bar is https://test.hyland.com/identity/user-groups/12345678-abcd-1234-abcde-12345789abc, the GUID you should copy is 12345678-abcd-1234-abcde-12345789abc.
5.  Update the DocSecurityType_null runtime property with the copied GUID. See Updating Runtime Properties for more information.
6.  Repeat these steps for the other security types (Guidewire PC Doc Security - Sensitive and Guidewire CC Doc Security - Unrestricted), matching the name of the user group with the key in the mappings file.
7.  Open the Studio Modeler application in the Hyland Content Innovation Cloud.
8.  Click on the Guidewire ClaimCenter Workspace project. The project window is displayed.
9.  Click the Upload doc to Guidewire CC process in the Process Automation pane on the left. The project page is displayed.
10.  Click Edit Process Variables in the Properties pane on the right. The Process variables page is displayed.
11.  Locate the docSecurityGuidNone variable and click the corresponding Edit variable button. The Edit variable page is displayed.
12.  In the Default value section, enter the GUID for the user group copied in step 4 in the Provide a value field.
13.  Click Save. The Process variables page is displayed.
14.  Locate the docSecurityGuidSensitive variable and click the corresponding Edit variable button. The Edit variable page is displayed.
15.  In the Default value section, enter the GUID for the user group copied in step 4 in the Provide a value field.
16.  Click Save. The Process variables page is displayed.
17.  Locate the docSecurityGuidUnrestricted variable and click the corresponding Edit variable button. The Edit variable page is displayed.
18.  In the Default value section, enter the GUID for the user group copied in step 4 in the Provide a value field.
19.  Click Save. The Process variables page is displayed.
20.  Click Save (the cloud icon at the top bar of the pane) to save the project.
21.  Select the Release action from the bar above the project window to release a new version of the project.
22.  Open the Studio Admin application.
23.  Navigate to Devops > Application Instances. The Application Instances page is displayed.
24.  Find your Guidewire ClaimCenter Workspace project and click Upgrade to upgrade it and update the variable in your application instance. See the Hyland Automate Modeler documentation for general information on upgrading a project.
